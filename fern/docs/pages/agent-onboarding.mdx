This page shows the full lifecycle: account creation, email verification, Stripe setup, listing creation, and sharing the checkout URL. Each step shows the agent–human conversation alongside the API call that drives it.

This is the canonical pattern for an agent operating ListBee on behalf of a human.

---

## Overview

Eight steps. Four require human input. Four are pure API calls the agent makes autonomously.

| Step | Who acts | What happens |
|------|----------|-------------|
| 1 | Agent | Asks human for email, calls `POST /v1/account/signup` |
| 2 | Human | Checks inbox, gives agent the 6-digit code |
| 3 | Agent | Calls `POST /v1/account/verify`, receives API key |
| 4 | Agent | Reads `readiness.operational` — it's `false` |
| 5 | Human | Provides Stripe key (or goes through Connect onboarding) |
| 6 | Agent | Submits Stripe key or sends Connect URL, polls until `operational: true` |
| 7 | Agent | Creates listing via `POST /v1/listings` |
| 8 | Agent | Shares checkout URL with human |

---

## Step 1: Get email, request OTP

The agent asks the human for their email address, then fires a signup request. No API key is needed — this is an unauthenticated call.

**Agent says:**
> What's your email address? I'll send you a verification code.

**Human says:**
> alex@example.com

**Agent calls:**

```python
import httpx

BASE = "https://api.listbee.so"

resp = httpx.post(
    f"{BASE}/v1/account/signup",
    json={"email": "alex@example.com"},
)
# 200 OK — OTP sent to alex@example.com
```

---

## Step 2: Human provides the code

The OTP is a 6-digit code sent to the email address. The agent cannot retrieve it — the human must relay it.

**Agent says:**
> Check your inbox. There's a 6-digit code from ListBee. What is it?

**Human says:**
> 847213

---

## Step 3: Verify and get the API key

The agent submits the email and code. The response contains the API key and the account object — including the initial readiness state.

```python
resp = httpx.post(
    f"{BASE}/v1/account/verify",
    json={"email": "alex@example.com", "code": "847213"},
)
data = resp.json()

api_key = data["api_key"]     # lb_... — store this
account = data["account"]     # account object with readiness

print(api_key)                # lb_live_...
print(account["id"])          # acc_...
print(account["readiness"]["operational"])  # False — Stripe not connected yet
```

The agent stores the API key. All subsequent requests use `Authorization: Bearer lb_...`.

---

## Step 4: Read readiness

After verification, `readiness.operational` is `false`. The account cannot process payments until Stripe is connected. The agent reads `readiness.next` to know what action to take.

```python
readiness = account["readiness"]

print(readiness["operational"])  # False
print(readiness["next"])         # "set_stripe_key" or "connect_stripe"

next_code = readiness["next"]
action = next(a for a in readiness["actions"] if a["code"] == next_code)

print(action["kind"])     # "api" if agent can resolve it, "human" if not
print(action["message"])  # "Submit your Stripe secret key to enable payments."
```

Two possible states at this point:

| `next` code | `kind` | Meaning |
|------------|--------|---------|
| `set_stripe_key` | `api` | Agent can submit a Stripe key directly |
| `connect_stripe` | `human` | Human must complete Stripe Connect in browser |

---

## Step 5: Stripe setup — two paths

### Path A: Human already has a Stripe key

**Agent says:**
> To accept payments, I need your Stripe secret key. Do you have one? It starts with `sk_live_` or `sk_test_`.

**Human says:**
> Yes — sk_live_51abc...

**Agent calls:**

```python
resp = httpx.post(
    f"{BASE}/v1/account/stripe-key",
    headers={"Authorization": f"Bearer {api_key}"},
    json={"secret_key": "sk_live_51abc..."},
)
account = resp.json()

print(account["readiness"]["operational"])  # True
```

### Path B: Human does not have a Stripe account

**Agent says:**
> No Stripe account yet? I'll generate a setup link. You'll need to complete it in your browser — takes about 2 minutes.

**Agent calls:**

```python
resp = httpx.post(
    f"{BASE}/v1/account/stripe/connect",
    headers={"Authorization": f"Bearer {api_key}"},
)
data = resp.json()

connect_url = data["url"]
print(connect_url)  # https://connect.stripe.com/setup/...
```

**Agent says:**
> Open this link to connect Stripe: https://connect.stripe.com/setup/...
> Come back here when you're done and I'll check if everything is ready.

---

## Step 6: Poll until operational

After the human completes Stripe Connect, the agent polls `GET /v1/account` until `readiness.operational` is `true`.

```python
import time

while True:
    resp = httpx.get(
        f"{BASE}/v1/account",
        headers={"Authorization": f"Bearer {api_key}"},
    )
    account = resp.json()

    if account["readiness"]["operational"]:
        print("Stripe connected — ready to sell.")
        break

    next_action = account["readiness"]["next"]
    print(f"Still waiting... next action: {next_action}")
    time.sleep(5)
```

Once `operational` is `true`, the account can process payments.

---

## Step 7: Create the listing

The agent collects the listing details from the human — or generates them from context — and calls `POST /v1/listings`.

**Agent says:**
> What would you like to sell? Give me a name, price, and the file or URL you want to deliver to buyers.

**Human says:**
> "Python Async Patterns" — $19, here's the PDF: https://my-bucket.s3.amazonaws.com/python-async.pdf

**Agent calls:**

```python
resp = httpx.post(
    f"{BASE}/v1/listings",
    headers={"Authorization": f"Bearer {api_key}"},
    json={
        "name": "Python Async Patterns",
        "price": 1900,          # cents
        "content": "https://my-bucket.s3.amazonaws.com/python-async.pdf",
    },
)
listing = resp.json()

print(listing["url"])                   # https://buy.listbee.so/r7kq2xy9
print(listing["readiness"]["sellable"]) # True
```

The `url` field is the product page. When `readiness.sellable` is `true`, buyers can purchase immediately.

---

## Step 8: Share the checkout URL

**Agent says:**
> Your listing is live. Share this URL with buyers:
> https://buy.listbee.so/r7kq2xy9
>
> When someone pays, they receive the PDF automatically by email.

## Full flow — compact version

```python
import httpx
import time

BASE = "https://api.listbee.so"

# Step 1: Request OTP
email = input("Enter your email: ")
httpx.post(f"{BASE}/v1/account/signup", json={"email": email})

# Step 2: Human provides code
code = input("Enter the 6-digit code from your inbox: ")

# Step 3: Verify
data = httpx.post(
    f"{BASE}/v1/account/verify",
    json={"email": email, "code": code},
).json()

api_key = data["api_key"]
account = data["account"]
headers = {"Authorization": f"Bearer {api_key}"}

# Step 4-6: Stripe setup
if not account["readiness"]["operational"]:
    next_code = account["readiness"]["next"]

    if next_code == "set_stripe_key":
        stripe_key = input("Enter your Stripe secret key (sk_live_...): ")
        httpx.post(
            f"{BASE}/v1/account/stripe-key",
            headers=headers,
            json={"secret_key": stripe_key},
        )
    elif next_code == "connect_stripe":
        url = httpx.post(
            f"{BASE}/v1/account/stripe/connect",
            headers=headers,
        ).json()["url"]
        print(f"Open this link to connect Stripe: {url}")
        input("Press Enter when done...")

    # Poll until operational
    while True:
        account = httpx.get(f"{BASE}/v1/account", headers=headers).json()
        if account["readiness"]["operational"]:
            break
        time.sleep(5)

# Step 7: Create listing
listing = httpx.post(
    f"{BASE}/v1/listings",
    headers=headers,
    json={
        "name": "Python Async Patterns",
        "price": 1900,
        "content": "https://my-bucket.s3.amazonaws.com/python-async.pdf",
    },
).json()

# Step 8: Share URL
print(f"Your listing is live: {listing['url']}")
```
