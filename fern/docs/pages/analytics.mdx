---
title: Google Analytics tracking
description: Inject GA4 on your product pages and track purchases automatically.
---

Set a GA4 Measurement ID on your account and ListBee injects `gtag.js` on every product page and thank-you page it hosts. No code changes required on your end.

When a buyer completes a purchase, ListBee fires a `purchase` event with `transaction_id`, `value`, and `currency`. UTM parameters flow through to GA4 campaign attribution.

<Note>
GA4 tracking is opt-in. If `ga_measurement_id` is not set, no GA script is injected — zero impact on existing sellers.
</Note>

---

## Set your Measurement ID

`PUT /v1/account` accepts `ga_measurement_id`. The format must match `G-XXXXXXXXXX` (where X is alphanumeric). Pass `null` to clear it.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.put(
    "https://api.listbee.so/v1/account",
    headers={"Authorization": "Bearer lb_..."},
    json={"ga_measurement_id": "G-ABC123XYZ"},
)
account = resp.json()

print(account["ga_measurement_id"])  # G-ABC123XYZ
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const account = await client.account.update({
  ga_measurement_id: "G-ABC123XYZ",
});

console.log(account.ga_measurement_id); // G-ABC123XYZ
```

</CodeGroup>

To remove tracking, pass `null`:

<CodeGroup>

```python title="Python"
httpx.put(
    "https://api.listbee.so/v1/account",
    headers={"Authorization": "Bearer lb_..."},
    json={"ga_measurement_id": None},
)
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

await client.account.update({ ga_measurement_id: null });
```

</CodeGroup>

## What gets injected

When `ga_measurement_id` is set, ListBee injects the standard `gtag.js` snippet on two pages:

- **Product page** — the checkout page buyers see when they click your listing URL
- **Thank-you page** — shown after successful payment

On the thank-you page, ListBee automatically fires a `purchase` event:

```json
{
  "transaction_id": "ord_...",
  "value": 19.00,
  "currency": "USD"
}
```

Nothing else is required. No script tags, no third-party plugins.

---

## UTM parameters

Each listing has three optional UTM fields: `utm_source`, `utm_medium`, and `utm_campaign`. They control the campaign attribution passed to GA4 when a buyer lands on the product page.

**Defaults (applied at render time if fields are null):**

| Field | Default |
|-------|---------|
| `utm_source` | `listbee` |
| `utm_medium` | `product_page` |
| `utm_campaign` | Listing slug |

You only need to set UTM fields if you want to override the defaults. The fields are stored as `null` on the listing when using defaults — the actual values are applied when the page renders.

### Create a listing with custom UTM fields

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Python Async Patterns",
        "price": 1900,
        "content": "https://my-bucket.s3.amazonaws.com/python-async.pdf",
        "utm_source": "newsletter",
        "utm_medium": "email",
        "utm_campaign": "march-2026-launch",
    },
)
listing = resp.json()

print(listing["utm_source"])    # newsletter
print(listing["utm_medium"])    # email
print(listing["utm_campaign"])  # march-2026-launch
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Python Async Patterns",
  price: 1900,
  content: "https://my-bucket.s3.amazonaws.com/python-async.pdf",
  utm_source: "newsletter",
  utm_medium: "email",
  utm_campaign: "march-2026-launch",
});

console.log(listing.utm_source);    // newsletter
console.log(listing.utm_medium);    // email
console.log(listing.utm_campaign);  // march-2026-launch
```

</CodeGroup>

### Create a listing using defaults

Omit the UTM fields entirely. ListBee uses `listbee` / `product_page` / `<slug>` at render time.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(

print(listing["utm_source"])    # None
print(listing["utm_medium"])    # None
print(listing["utm_campaign"])  # None
```

```typescript title="TypeScript"
import { ListBee } from "listbee";
```

</CodeGroup>

### Update UTM fields on an existing listing

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.put(
    "https://api.listbee.so/v1/listings/python-async-patterns",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "utm_source": "twitter",
        "utm_medium": "social",
        "utm_campaign": "april-promo",
    },
)
listing = resp.json()
print(listing["utm_campaign"])  # april-promo
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.update("python-async-patterns", {
  utm_source: "twitter",
  utm_medium: "social",
  utm_campaign: "april-promo",
});
console.log(listing.utm_campaign); // april-promo
```

</CodeGroup>

## Field reference

### Account

| Field | Type | Description |
|-------|------|-------------|
| `ga_measurement_id` | `string \| null` | GA4 Measurement ID. Format: `G-XXXXXXXXXX`. Set to `null` to disable tracking. |

### Listing (create and update)

| Field | Type | Max length | Description |
|-------|------|-----------|-------------|
| `utm_source` | `string \| null` | 100 | UTM source sent to GA4. Defaults to `listbee`. |
| `utm_medium` | `string \| null` | 100 | UTM medium sent to GA4. Defaults to `product_page`. |
| `utm_campaign` | `string \| null` | 100 | UTM campaign sent to GA4. Defaults to the listing slug. |
