All ListBee API requests authenticate with an API key passed as a Bearer token. Keys have the prefix `lb_` and are tied to a [store](/stores).

## Two ways to get a key

1. **Bootstrap API** (agent-driven) — the agent creates a store and gets a key via 3 API calls. A human provides their email and an OTP code.
2. **Console** (human-driven) — sign in at [console.listbee.so](https://console.listbee.so). A default store and key are created on first login.

Both paths produce the same result: a store with an API key.

## Bootstrap flow

Three steps, one focus each. The agent drives the flow, the human verifies their email.

### Step 1: Send OTP

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import ListBee

result = ListBee.bootstrap.start(email="seller@example.com")
session = result.session
# Ask the human: "Check your email for a 6-digit code"
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/bootstrap \
  -H "Content-Type: application/json" \
  -d '{"email": "seller@example.com"}'
```
</CodeBlock>
</CodeBlocks>

Response:

```json
{
  "object": "bootstrap_session",
  "session": "sess_abc123...",
  "otp_sent": true
}
```

### Step 2: Verify OTP

The human reads the 6-digit code from their email. The agent passes it back.

<CodeBlocks>
<CodeBlock title="Python">
```python
result = ListBee.bootstrap.verify(session=session, code="847291")
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/bootstrap/verify \
  -H "Content-Type: application/json" \
  -d '{"session": "sess_abc123...", "code": "847291"}'
```
</CodeBlock>
</CodeBlocks>

If the email is new, an account is created. If it already exists, the existing account is used. The agent doesn't know which case occurred.

### Step 3: Create store

<CodeBlocks>
<CodeBlock title="Python">
```python
store = ListBee.bootstrap.create_store(
    session=session,
    store_name="SEO Pro"
)
api_key = store.api_key  # lb_... — shown once, save it
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/bootstrap/store \
  -H "Content-Type: application/json" \
  -d '{"session": "sess_abc123...", "store_name": "SEO Pro"}'
```
</CodeBlock>
</CodeBlocks>

Response:

```json
{
  "object": "store",
  "id": "st_7kQ2xY9mN3pR5tW1vB8a",
  "api_key": "lb_abc123...",
  "display_name": "SEO Pro",
  "slug": "seo-pro",
  "url": "https://buy.listbee.so/seo-pro",
  "readiness": {
    "sellable": false,
    "actions": [{ "code": "connect_stripe", "kind": "human", "message": "Connect Stripe to accept payments" }],
    "next": "connect_stripe"
  }
}
```

The `api_key` is returned once. Store it and use it as `Authorization: Bearer lb_...` for all subsequent requests.

## Using a key

```
Authorization: Bearer lb_...
```

Each key belongs to one store. All operations through that key are scoped to that store's listings, orders, and webhooks.

Missing or invalid keys return `401`.

## Same email, second store

An agent bootstrapping with an email that already has an account creates a new store on that account. If Stripe is already connected, the new store is immediately sellable.

## Key rotation

Keys can be rotated in the [console](https://console.listbee.so). The old key is immediately invalidated. The store ID stays the same.
