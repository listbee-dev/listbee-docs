---
title: Authentication
description: How to authenticate with the ListBee API and manage your API keys.
---

All API requests require a Bearer token in the `Authorization` header. Use the API key you receive during account setup.

```
Authorization: Bearer lb_...
```

Every request must include this header. Requests without it return `401 authentication_required`.

---

## Create an account

Account creation is a two-step flow: request a one-time code, then verify it to receive your API key.

<CodeGroup>

```python title="Python"
import httpx

BASE = "https://api.listbee.so"

# Step 1: request OTP
httpx.post(
    f"{BASE}/v1/account/signup",
    json={"email": "you@example.com"},
)
# 200 OK — 6-digit code sent to you@example.com

# Step 2: verify and receive API key
resp = httpx.post(
    f"{BASE}/v1/account/verify",
    json={"email": "you@example.com", "code": "123456"},
)
data = resp.json()

api_key = data["api_key"]    # lb_... — store securely
account_id = data["account"]["id"]  # acc_...
```

```typescript title="TypeScript"
const BASE = "https://api.listbee.so";

// Step 1: request OTP
await fetch(`${BASE}/v1/account/signup`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ email: "you@example.com" }),
});
// 200 OK — 6-digit code sent to you@example.com

// Step 2: verify and receive API key
const resp = await fetch(`${BASE}/v1/account/verify`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ email: "you@example.com", code: "123456" }),
});
const data = await resp.json();

const apiKey = data.api_key;           // lb_... — store securely
const accountId = data.account.id;     // acc_...
```

</CodeGroup>

The `api_key` in the `POST /v1/account/verify` response is shown **once**. Store it immediately. If you lose it, create a new key.

## API key management

### Create a key

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/api-keys",
    headers={"Authorization": "Bearer lb_..."},
    json={"name": "production"},
)
key = resp.json()

print(key["id"])       # lbk_...
print(key["key"])      # lb_... — shown once, store now
print(key["name"])     # production
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/api-keys", {
  method: "POST",
  headers: {
    "Authorization": "Bearer lb_...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ name: "production" }),
});
const key = await resp.json();

console.log(key.id);    // lbk_...
console.log(key.key);   // lb_... — shown once, store now
console.log(key.name);  // production
```

</CodeGroup>

### List keys

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.get(
    "https://api.listbee.so/v1/api-keys",
    headers={"Authorization": "Bearer lb_..."},
)
result = resp.json()

for key in result["data"]:
    print(key["id"], key["name"], key["prefix"])
    # lbk_...   production   lb_live_abc
    # The full key value is NOT returned — prefix only.
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/api-keys", {
  headers: { "Authorization": "Bearer lb_..." },
});
const result = await resp.json();

for (const key of result.data) {
  console.log(key.id, key.name, key.prefix);
  // lbk_...   production   lb_live_abc
  // The full key value is NOT returned — prefix only.
}
```

</CodeGroup>

### Delete a key

<CodeGroup>

```python title="Python"
import httpx

httpx.delete(
    "https://api.listbee.so/v1/api-keys/lbk_...",
    headers={"Authorization": "Bearer lb_..."},
)
# 204 No Content — key is revoked immediately
```

```typescript title="TypeScript"
await fetch("https://api.listbee.so/v1/api-keys/lbk_...", {
  method: "DELETE",
  headers: { "Authorization": "Bearer lb_..." },
});
// 204 No Content — key is revoked immediately
```

</CodeGroup>

Deletion is immediate. Any in-flight requests using the deleted key will be rejected.

## Key rotation

Rotate API keys without downtime:

1. Create a new key via `POST /v1/api-keys`.
2. Update your application config to use the new key.
3. Delete the old key via `DELETE /v1/api-keys/{id}`.

There is no grace period — the old key stops working as soon as it is deleted.

---

## Security

- **Keys are hashed at rest.** ListBee stores only a hash of your key, not the plaintext value.
- **Shown once.** The full key value is returned only in the `POST /v1/api-keys` response. After that, only the prefix (`lb_live_abc`) is visible in list responses.
- **Prefix-only in list.** `GET /v1/api-keys` returns `prefix`, not `key`. Use the prefix to identify which key to rotate or delete.
- **Treat keys as secrets.** Do not commit them to source control. Use environment variables or a secrets manager.

#

# Key is shown once on creation; list returns prefix only
# Keys are hashed at rest; deletion is immediate
#
# Rotation: create new → update config → delete old
#
# Errors: RFC 9457 { type, title, status, detail, code, param }
