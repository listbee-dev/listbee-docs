All ListBee API requests authenticate with an API key passed as a Bearer token. Keys have the prefix `lb_` and are tied to an account.

## Two ways to get a key

1. **Bootstrap API** (agent-driven) — the agent creates an account and gets a key via 3 API calls. A human provides their email and an OTP code.
2. **Console** (human-driven) — sign in at [console.listbee.so](https://console.listbee.so). An account and key are created on first login.

Both paths produce the same result: an account with an API key.

## Bootstrap flow

Three steps, one focus each. The agent drives the flow, the human verifies their email.

### Step 1: Send OTP

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import ListBee

client = ListBee()  # no api_key needed for bootstrap
result = client.bootstrap.start(email="seller@example.com")
session = result.session
# Ask the human: "Check your email for a 6-digit code"
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/bootstrap \
  -H "Content-Type: application/json" \
  -d '{"email": "seller@example.com"}'
```
</CodeBlock>
</CodeBlocks>

Response:

```json
{
  "object": "bootstrap_session",
  "session": "sess_abc123...",
  "otp_sent": true
}
```

### Step 2: Verify OTP

The human reads the 6-digit code from their email. The agent passes it back.

<CodeBlocks>
<CodeBlock title="Python">
```python
result = client.bootstrap.verify(session=session, code="847291")
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/bootstrap/verify \
  -H "Content-Type: application/json" \
  -d '{"session": "sess_abc123...", "code": "847291"}'
```
</CodeBlock>
</CodeBlocks>

If the email is new, an account is created. If it already exists, the existing account is returned. The agent doesn't need to know which case occurred.

### Step 3: Complete bootstrap

<CodeBlocks>
<CodeBlock title="Python">
```python
result = client.bootstrap.complete(session=session)
api_key = result.api_key  # lb_... — shown once, save it
account_id = result.account_id  # acc_...
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/bootstrap/complete \
  -H "Content-Type: application/json" \
  -d '{"session_id": "sess_abc123..."}'
```
</CodeBlock>
</CodeBlocks>

Response:

```json
{
  "object": "bootstrap",
  "account_id": "acc_7kQ2xY9mN3pR5tW1vB8a",
  "api_key": "lb_abc123..."
}
```

The `api_key` is returned once. Save it and use it as `Authorization: Bearer lb_...` for all subsequent requests.

## Using a key

```
Authorization: Bearer lb_...
```

Each key belongs to one account. All operations through that key are scoped to that account's listings, orders, and webhooks.

Missing or invalid keys return `401`.

## Same email, existing account

Bootstrapping with an email that already has an account returns the existing account. If Stripe was already connected, the account is immediately sellable.

## Key rotation

Keys can be rotated in the [console](https://console.listbee.so). The old key is immediately invalidated. The account ID stays the same.
