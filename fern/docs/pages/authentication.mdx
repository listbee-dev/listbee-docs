All ListBee API requests authenticate with an API key passed as a Bearer token. Keys have the prefix `lb_` and are scoped to a single account.

## How it works

API keys are created in the [developer console](https://console.listbee.so) or via the API. The console uses email OTP login — no password, no signup endpoint. Once you have an account, every API request carries your key in the `Authorization` header.

Keys are permanent until deleted. There is no expiry. You can create multiple keys — one per agent, integration, or environment — and revoke individual keys without disrupting others.

The full key value is returned once at creation. ListBee does not store it in retrievable form. If you lose a key, delete it and create a new one.

## Getting your first key

Log in at [console.listbee.so](https://console.listbee.so) with your email. An OTP is sent to your inbox. After logging in, your first API key is shown on the dashboard.

## Creating additional keys

<CodeBlocks>
  <CodeBlock title="Python">
```python
from listbee import ListBee

client = ListBee(api_key="lb_...")

key = client.api_keys.create(name="n8n integration")

print(key.id)     # lbk_7kQ2xY9mN3pR5tW1vB8a
print(key.key)    # lb_... (shown once — store it now)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/api-keys \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "n8n integration"}'
```
  </CodeBlock>
</CodeBlocks>

Response:

```json
{
  "object": "api_key",
  "id": "lbk_7kQ2xY9mN3pR5tW1vB8a",
  "name": "n8n integration",
  "prefix": "lb_7kQ2xY",
  "key": "lb_7kQ2xY9mN3pR5tW1vB8a01...",
  "created_at": "2026-04-09T10:00:00Z"
}
```

The `key` field contains the full value. This is the only time it is returned.

## Using a key

Pass the key in the `Authorization` header on every request:

```
Authorization: Bearer lb_...
```

Missing or invalid keys return `401`. Using a deleted key returns `401`.

## Listing keys

<CodeBlocks>
  <CodeBlock title="Python">
```python
keys = client.api_keys.list()

for key in keys.data:
    print(key.id, key.name, key.prefix)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl https://api.listbee.so/v1/api-keys \
  -H "Authorization: Bearer lb_..."
```
  </CodeBlock>
</CodeBlocks>

The response includes full key values for all keys. Do not log or transmit this response.

## Deleting a key

<CodeBlocks>
  <CodeBlock title="Python">
```python
client.api_keys.delete(key_id="lbk_7kQ2xY9mN3pR5tW1vB8a")
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X DELETE https://api.listbee.so/v1/api-keys/lbk_7kQ2xY9mN3pR5tW1vB8a \
  -H "Authorization: Bearer lb_..."
```
  </CodeBlock>
</CodeBlocks>

You cannot delete the key used in the current request. Create and switch to a replacement key first.

## Key fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | API key entity ID (`lbk_` prefixed) |
| `name` | string | Human-readable label |
| `prefix` | string | First 8 characters of the key value |
| `key` | string | Full key value — returned at creation only |
| `created_at` | string | ISO 8601 timestamp |
