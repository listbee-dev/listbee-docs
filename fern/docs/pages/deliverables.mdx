---
title: Deliverables
description: How ListBee detects and delivers digital content — files, URLs, and text.
---

A deliverable is the digital content attached to a listing. ListBee detects the type from what you pass to the `content` field and handles delivery automatically when using [managed fulfillment](/fulfillment-modes).

Deliverables are one axis of the fulfillment model. The other is the [fulfillment mode](/fulfillment-modes) (managed vs external).

| Deliverable type | Fulfillment mode | Result |
|-----------------|-----------------|--------|
| FILE / URL / TEXT | Managed | ListBee delivers automatically |
| FILE / URL / TEXT | External | Content stored but your app handles delivery |
| None | External | No content on listing — your app delivers everything |

---

## Detection logic

When you set `content`, ListBee inspects the value:

1. If the value is not a URL → **TEXT**
2. If the value is a URL → ListBee sends a `HEAD` request
3. Response `Content-Type` is a file MIME type (e.g. `application/pdf`, `application/zip`) → **FILE**
4. Response `Content-Type` is `text/html` → **URL**

The detected type is stored as `deliverable_type` on the listing: `"file"`, `"url"`, or `"text"`.

<Tip>
Detection is automatic. You don't need to specify the type in most cases.
</Tip>

---

## FILE

A URL that points to a downloadable file.

**Examples:** PDF, ZIP, EPUB, MP3, dataset, software binary.

**Delivery:** ListBee fetches the file, stores it on a CDN, and generates a signed time-limited download URL. The buyer receives an email with the link. Expires after 24 hours.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Advanced Python Patterns",
        "price": 2900,
        "content": "https://my-bucket.s3.amazonaws.com/advanced-python.pdf",
    },
)
listing = resp.json()

print(listing["deliverable_type"])  # file
print(listing["fulfillment"])       # managed
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Advanced Python Patterns",
  price: 2900,
  content: "https://my-bucket.s3.amazonaws.com/advanced-python.pdf",
});

console.log(listing.deliverable_type);  // file
console.log(listing.fulfillment);       // managed
```

</CodeGroup>

## URL

A URL that points to a web page.

**Examples:** Notion page, Google Doc, gated article, course login page.

**Delivery:** After payment, the buyer receives an email with a link. Clicking it redirects to your URL. ListBee does not cache the page content.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Private Research Notes",
        "price": 1500,
        "content": "https://www.notion.so/your-page-id",
    },
)
listing = resp.json()
print(listing["deliverable_type"])  # url
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Private Research Notes",
  price: 1500,
  content: "https://www.notion.so/your-page-id",
});
console.log(listing.deliverable_type);  // url
```

</CodeGroup>

## TEXT

Plain text — a license key, API key, password, code snippet, discount code.

**Delivery:** After payment, the text is shown on the confirmation page and included in the buyer email.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},

        "name": "Premium API Access",
        "price": 4900,
        "content": "LICENSE-KEY-ABCD-1234-EFGH-5678",
    },
)
listing = resp.json()
print(listing["deliverable_type"])  # text
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

```

## No deliverable (external only)

External fulfillment listings don't need a deliverable. Set `fulfillment: "external"` without `content` — your app handles all delivery.

```python
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Custom Portrait Drawing",
        "price": 4900,
        "fulfillment": "external",
        # no content — deliverable_type will be null
    },
)
listing = resp.json()
print(listing["deliverable_type"])  # None
print(listing["fulfillment"])       # external
```

## Renamed fields

The `content_type` field has been renamed to `deliverable_type`. Similarly:

| Old field | New field |
|-----------|-----------|
| `content_type` | `deliverable_type` |
| `content_key` | `deliverable_key` |
| `content_value` | `deliverable_value` |
| `content_meta` | `deliverable_meta` |
| `has_content` | `has_deliverable` |

The `content` field on create/update requests is unchanged — you still pass content the same way.

---

## Summary

| Type | `content` value | After purchase (managed) |
|------|----------------|--------------------------|
| `file` | URL → file MIME type | Signed CDN download URL, expires 24h |
| `url` | URL → HTML page | Redirect to your URL |
| `text` | Non-URL string | Text shown inline in email |
| `null` | No content (external only) | Your app delivers |
