---
title: Dynamic fulfillment
description: Generate content after payment and push it to ListBee for delivery — AI agents, on-demand services, custom generation.
---

Dynamic fulfillment bridges external and managed delivery. Your app receives the `order.paid` webhook, generates content (AI output, custom report, personalized file), then calls `POST /v1/orders/{id}/fulfill` to push it back to ListBee. ListBee delivers the generated content to the buyer — same secure access page, same delivery email — as if you had attached the content to the listing from the start.

---

## How it works

```
Buyer pays on product page
        ↓
ListBee fires order.paid webhook (includes checkout_data)
        ↓
Your agent generates content using checkout_data as input
        ↓
Agent calls POST /v1/orders/{id}/fulfill with deliverable
        ↓
ListBee stores content + creates access grant
        ↓
Buyer receives delivery email with secure access link
        ↓
ListBee fires order.fulfilled webhook
```

The buyer gets the same delivery experience as managed fulfillment — email with download link, redirect, or inline text — but the content was generated on-demand by your code.

## Setup

Create a listing with `fulfillment: "external"` and a `checkout_schema` to collect buyer inputs. These inputs arrive in `checkout_data` on the `order.paid` webhook, ready for your generation logic.

<CodeGroup>

```python title="Python"
import listbee

lb = listbee.ListBee(api_key="lb_prod_...")

listing = lb.listings.create(
    name="Personal Horoscope Reading",
    price=500,
    fulfillment="external",
    checkout_schema=[
        {
            "key": "birth_date",
            "type": "date",
            "label": "Your birth date",
            "required": True,
        },
        {
            "key": "zodiac_focus",
            "type": "select",
            "label": "Reading focus",
            "required": False,
            "options": ["Career", "Love", "Health", "General"],
        },
    ],
)

print(listing.url)         # https://buy.listbee.so/ai-horoscope
print(listing.fulfillment) # external
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_prod_..." });

const listing = await client.listings.create({
  name: "Personal Horoscope Reading",
  price: 500,
  fulfillment: "external",
  checkoutSchema: [
    {
      key: "birth_date",
      type: "date",
      label: "Your birth date",
      required: true,
    },
    {
      key: "zodiac_focus",
      type: "select",
      label: "Reading focus",
      required: false,
      options: ["Career", "Love", "Health", "General"],
    },
  ],
});

console.log(listing.url);         // https://buy.listbee.so/ai-horoscope
console.log(listing.fulfillment); // external
```

## Walkthrough: AI horoscope agent

An AI agent sells personalized horoscopes. The buyer provides their birth date at checkout. After payment, the agent generates a horoscope and pushes it to ListBee for delivery.

### Step 1: Handle the webhook and generate content

<CodeGroup>

```python title="Python"
import hashlib
import hmac
import listbee
from fastapi import Request, HTTPException

WEBHOOK_SECRET = "whsec_..."
lb = listbee.ListBee(api_key="lb_prod_...")

async def handle_webhook(request: Request):
    payload = await request.body()
    signature = request.headers.get("X-ListBee-Signature", "")

    expected = hmac.new(WEBHOOK_SECRET.encode(), payload, hashlib.sha256).hexdigest()
    if not hmac.compare_digest(expected, signature):
        raise HTTPException(status_code=400, detail="Invalid signature")

    event = await request.json()

    if event["type"] == "order.paid":
        order = event["data"]["object"]
        birth_date = order["checkout_data"]["birth_date"]
        focus = order["checkout_data"].get("zodiac_focus", "General")

        # Generate the horoscope (your AI logic here)
        horoscope = generate_horoscope(birth_date, focus)

        # Push generated content to ListBee for delivery
        lb.orders.fulfill(order["id"], deliverable=horoscope)

    return {"ok": True}

def generate_horoscope(birth_date: str, focus: str) -> str:
    # Your AI generation logic
    return f"Your personalized {focus.lower()} horoscope for {birth_date}..."
```

```typescript title="TypeScript"
import { createHmac, timingSafeEqual } from "crypto";
import { ListBee } from "listbee";
import type { Request, Response } from "express";

const WEBHOOK_SECRET = "whsec_...";
const client = new ListBee({ apiKey: "lb_prod_..." });

export async function handleWebhook(req: Request, res: Response) {
  const payload = req.body as Buffer;
  const signature = req.headers["x-listbee-signature"] as string;

  const expected = createHmac("sha256", WEBHOOK_SECRET)
    .update(payload)
    .digest("hex");

  const sigBuf = Buffer.from(signature, "hex");
  const expBuf = Buffer.from(expected, "hex");

  if (sigBuf.length !== expBuf.length || !timingSafeEqual(sigBuf, expBuf)) {
    return res.status(400).json({ error: "Invalid signature" });
  }

  const event = JSON.parse(payload.toString());

  if (event.type === "order.paid") {
    const order = event.data.object;
    const birthDate = order.checkout_data.birth_date;
    const focus = order.checkout_data.zodiac_focus ?? "General";

    // Generate the horoscope (your AI logic here)
    const horoscope = generateHoroscope(birthDate, focus);

    // Push generated content to ListBee for delivery
    await client.orders.fulfill(order.id, { deliverable: horoscope });
  }

  res.json({ ok: true });
}

function generateHoroscope(birthDate: string, focus: string): string {
  // Your AI generation logic
  return `Your personalized ${focus.toLowerCase()} horoscope for ${birthDate}...`;
}
```

</CodeGroup>

### Step 2: Buyer receives the content

After your `fulfill` call, ListBee creates an access grant and delivers the generated content to the buyer by email. The buyer clicks the link and sees the content on a secure access page — the same experience as a static digital product.

---

## The fulfill endpoint

`POST /v1/orders/{id}/fulfill` accepts a single `deliverable` field. ListBee auto-detects the content type:

| `deliverable` value | Detected as | How buyer receives it |
|--------------------|--------------|-----------------------|
| Plain text string | `text` | Displayed inline on access page |
| `https://...` pointing to a file | `file` | Fetched and stored on CDN, download link sent |
| `https://...` pointing to a webpage | `url` | Stored as redirect, buyer redirected on access |

<CodeGroup>

```python title="Python — text content"
import listbee

lb = listbee.ListBee(api_key="lb_prod_...")

# Deliver generated text inline
order = lb.orders.fulfill(
    "ord_abc123",
    deliverable="Mercury is in retrograde. Today favors reflection over action...",
)
print(order.status)  # fulfilled
```

```python title="Python — file URL"
import listbee

lb = listbee.ListBee(api_key="lb_prod_...")

# Deliver a generated file (ListBee fetches and stores it on CDN)
order = lb.orders.fulfill(
    "ord_abc123",
    deliverable="https://your-bucket.s3.amazonaws.com/generated-report.pdf",
)
print(order.status)  # fulfilled
```

```typescript title="TypeScript — text content"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_prod_..." });

const order = await client.orders.fulfill("ord_abc123", {

```

## Use cases

| Use case | Checkout schema fields | Deliverable |
|----------|------------------------|-------------|
| AI horoscope reading | `birth_date` (date), `zodiac_focus` (select) | Generated text |
| Custom avatar | `style` (select), `description` (text) | Generated image URL |
| Personalized market report | `company_name` (text), `industry` (select) | Generated PDF URL |
| On-demand code review | `repo_url` (text) | Analysis markdown |
| Tailored meal plan | `dietary_restrictions` (text), `goal` (select) | Generated text or PDF |
