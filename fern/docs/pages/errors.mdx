All error responses use [RFC 9457 Problem Details](https://www.rfc-editor.org/rfc/rfc9457) with `Content-Type: application/problem+json`. Every error has the same shape — parse once, handle everywhere.

---

## Error format

```json
{
  "type": "https://docs.listbee.so/errors/validation-error",
  "title": "Validation error",
  "status": 422,
  "detail": "The price field must be a positive integer.",
  "code": "invalid_field",
  "param": "price"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | URI identifying the error type. Use this for programmatic matching. |
| `title` | string | Short human-readable summary of the error type. |
| `status` | integer | HTTP status code. Matches the response status. |
| `detail` | string | Specific description of what went wrong in this request. |
| `code` | string | Machine-readable error code. Use this to branch error handling logic. |
| `param` | string \| null | The field that caused the error, if applicable. `null` for non-field errors. |

---

## HTTP status codes

| Status | Name | When it occurs |
|--------|------|---------------|
| `400` | Bad request | The request body is malformed or missing required fields. |
| `401` | Unauthorized | No API key provided, or the key is invalid or revoked. |
| `404` | Not found | The resource does not exist, or you do not have access to it. |
| `409` | Conflict | The request conflicts with existing state (e.g. duplicate idempotency key with different body). |
| `422` | Unprocessable entity | The request is well-formed but fails validation (e.g. negative price, missing required field). |
| `429` | Too many requests | Rate limit exceeded. See the `Retry-After` header for when to retry. |
| `500` | Internal server error | Something went wrong on ListBee's side. Retrying with backoff is appropriate. |

---

## Common error codes

| Code | Cause | Resolution |
|------|-------|------------|
| `authentication_required` | No API key in the `Authorization` header | Add `Authorization: Bearer lb_...` header to the request |
| `invalid_api_key` | Key is malformed, expired, or revoked | Check key format (`lb_` prefix) or create a new key via `POST /v1/api-keys` |
| `not_found` | Resource does not exist | Verify the ID or slug — listings use slug, not ID |
| `invalid_field` | A field failed validation | Check the `param` field in the error response for which field, then fix the value |
| `missing_field` | A required field was not provided | Check the `param` field, then add the missing field to the request body |
| `unsupported_currency` | The currency code is not supported | Use a supported currency — currency is set at the account level |
| `price_too_low` | Price is below Stripe's minimum | Set price to at least 50 (smallest currency unit, e.g. 50 cents) |
| `stripe_not_connected` | No Stripe account linked to this account | Call `POST /v1/account/stripe-key` with your Stripe secret key |
| `content_fetch_failed` | ListBee couldn't reach the content URL | Verify the URL is publicly accessible and returns HTTP 200 |
| `idempotency_conflict` | Same `Idempotency-Key` sent with a different request body | Generate a new UUID for the new request |
| `rate_limit_exceeded` | Too many requests for this endpoint | Wait for `Retry-After` header value (seconds), then retry |
| `internal_error` | Unexpected server error | Safe to retry with exponential backoff — if persistent, contact support |

## Handling errors in code

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={"name": "My Guide", "price": -100},
)

if not resp.is_success:
    error = resp.json()
    code = error.get("code")
    param = error.get("param")

    if code == "invalid_field":
        print(f"Validation error on field '{param}': {error['detail']}")
    elif code == "authentication_required":
        print("Check your API key.")
    elif resp.status_code == 429:
        retry_after = resp.headers.get("Retry-After", "60")
        print(f"Rate limited. Retry after {retry_after} seconds.")
    else:
        print(f"Error {resp.status_code}: {error['detail']}")
```

```typescript title="TypeScript"
import { ListBee, ValidationError, AuthenticationError, RateLimitError, ListBeeError } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

try {
  const listing = await client.listings.create({
    name: "My Guide",
    price: -100,
  });
} catch (e) {
  if (e instanceof ValidationError) {
    console.error(`Validation error on field '${e.param}': ${e.detail}`);
  } else if (e instanceof AuthenticationError) {
    console.error("Check your API key.");
  } else if (e instanceof RateLimitError) {
    console.error(`Rate limited. Reset: ${e.reset}`);
  } else if (e instanceof ListBeeError) {
    console.error(`Error ${e.status}: ${e.detail}`);
  }
}
```
