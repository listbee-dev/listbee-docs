Build an n8n workflow that creates a listing wired to an `agent_callback_url`, then posts to Slack whenever an order is paid. Covers the two most common ListBee actions in n8n: creating content to sell and reacting to payments.

## Prerequisites

- n8n instance with `n8n-nodes-listbee` installed (see [n8n setup](/n8n))
- ListBee API key from [console.listbee.so](https://console.listbee.so) or [bootstrap](/authentication)
- Slack credential configured in n8n

## Workflow steps

### 1. ListBee Trigger — receive orders

Start with a **ListBee Trigger** node:

- **Events:** `order.paid`
- Copy the generated **Webhook URL** — you'll paste it into the listing below.

The trigger node fires whenever ListBee POSTs an event to this URL. The full payload is available in `$json`.

### 2. Create the listing (one-time setup)

In a separate workflow driven by a **Manual Trigger**, add a **ListBee** node:

- **Resource:** Listing
- **Operation:** Create
- **Name:** `Weekly Report Template`
- **Price:** `4900`
- **Fulfillment Mode:** `static`
- **Deliverable:** `{ "type": "url", "value": "https://example.com/weekly-report-template.pdf" }`
- **Agent Callback URL:** the trigger URL from step 1

<CodeBlocks>
<CodeBlock title="curl equivalent">
```bash
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Weekly Report Template",
    "price": 4900,
    "description": "A plug-and-play template for weekly business reviews.",
    "fulfillment_mode": "static",
    "deliverable": {"type": "url", "value": "https://example.com/weekly-report-template.pdf"},
    "agent_callback_url": "https://your-n8n.example.com/webhook/<trigger-id>"
  }'
```
</CodeBlock>
</CodeBlocks>

Save the `signing_secret` from the create response. The trigger node uses it to verify HMAC signatures on every incoming event.

### 3. Send a Slack message

Back in the trigger workflow, connect a **Slack** node after the ListBee Trigger:

- **Resource:** Message
- **Operation:** Post
- **Channel:** `#sales`
- **Text:**
  ```
  New order: {{ $json.data.buyer_email }} paid ${{ $json.data.amount / 100 }} for {{ $json.data.listing.name }}
  ```

## Order payload shape

When `order.paid` fires, the event body looks like:

```json
{
  "id": "evt_abc123",
  "type": "order.paid",
  "created": "2026-04-17T10:00:00Z",
  "api_version": "2026-04-17",
  "data": {
    "object": "order",
    "id": "ord_abc123",
    "status": "paid",
    "buyer_email": "buyer@example.com",
    "amount": 4900,
    "currency": "USD",
    "checkout_data": {},
    "listing": {"id": "lst_abc123", "name": "Weekly Report Template"},
    "unlock_url": "https://buy.listbee.so/unlock/...",
    "metadata": {}
  }
}
```

Use `$json.data.buyer_email`, `$json.data.amount`, and `$json.data.listing.name` in downstream nodes.

## Variations

**Filter by listing** — add an **IF** node after the trigger to route events from specific listings:

```
{{ $json.data.listing.id === 'lst_abc123' }}
```

**Async fulfillment** — set `fulfillment_mode: async` and leave `deliverable` null on create. After `order.paid`, generate content in n8n (LLM node, Google Sheets pull, etc.) and use the **ListBee** node with **Operation: Fulfill** to push the deliverable back.

**Reconcile after downtime** — if n8n was down when orders happened, use the **ListBee** node with **Operation: List** on the Event resource (pulling `GET /v1/events`) to backfill, or call **Operation: Redeliver** on an order to requeue its webhook.
