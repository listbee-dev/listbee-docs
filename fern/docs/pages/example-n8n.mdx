This example shows two n8n workflows: a simple digital product sale with Slack notifications, and a physical product workflow with address collection and shipping updates.

---

## Workflow 1: Digital product with notifications

### Workflow overview

| Node | Type | Purpose |
|------|------|---------|
| Manual trigger | Manual Trigger | Start the workflow to create a listing |
| Create listing | HTTP Request | `POST /v1/listings` |
| Order webhook | Webhook | Receive `order.paid` events from ListBee |
| Send Slack notification | Slack | Post to `#sales` when an order is paid |

### Node 1: Create listing

Use an **HTTP Request** node to call `POST /v1/listings`.

**Configuration:**

| Field | Value |
|-------|-------|
| Method | `POST` |
| URL | `https://api.listbee.so/v1/listings` |
| Authentication | Header Auth |
| Header name | `Authorization` |
| Header value | `Bearer lb_...` |
| Body type | JSON |

**Body:**

```json
{
  "name": "Python Async Patterns",
  "price": 1900,
  "deliverable": "https://my-bucket.s3.amazonaws.com/python-async.pdf"
}
```

**Response fields used downstream:**
- `{{ $json.url }}` — product page URL
- `{{ $json.slug }}` — listing slug
- `{{ $json.fulfillment }}` — `"managed"` or `"external"`
- `{{ $json.readiness.sellable }}` — whether buyers can purchase

### Node 2: Order webhook

Add a **Webhook** node to receive order events from ListBee.

**Configuration:**

| Field | Value |
|-------|-------|
| HTTP method | `POST` |
| Path | `/listbee-orders` (your choice) |
| Response mode | Immediately |

Copy the webhook URL from n8n (e.g. `https://your-n8n.example.com/webhook/listbee-orders`) and register it in ListBee:

```http
POST https://api.listbee.so/v1/webhooks
Authorization: Bearer lb_...
Content-Type: application/json

{
  "url": "https://your-n8n.example.com/webhook/listbee-orders",
  "events": ["order.paid", "order.fulfilled"]
}
```

Incoming `order.paid` payload:

```json
{
  "object": "event",
  "id": "evt_...",
  "type": "order.paid",
  "data": {
    "object": {
      "object": "order",
      "id": "ord_...",
      "status": "paid",
      "listing_id": "lst_...",
      "amount": 1900,
      "currency": "usd",
      "buyer_email": "buyer@example.com",
      "checkout_data": null,
      "shipping_address": null,
      "fulfillment_status": "pending",
      "paid_at": "2026-04-02T10:00:00Z",
      "created_at": "2026-04-02T09:59:00Z"
    }
  }
}
```

### Node 3: Slack notification

Add a **Slack** node after the webhook node.

**Configuration:**

| Field | Value |
|-------|-------|
| Resource | Message |
| Operation | Post |
| Channel | `#sales` |
| Text | `New order: {{ $json.data.object.buyer_email }} paid ${{ ($json.data.object.amount / 100).toFixed(2) }} for listing {{ $json.data.object.listing_id }}` |

## Workflow 2: Physical product with shipping

This workflow handles physical products — collecting a shipping address, processing the order, and updating shipping status.

### Workflow overview

| Node | Type | Purpose |
|------|------|---------|
| Create physical listing | HTTP Request | `POST /v1/listings` with external fulfillment + address |
| Order webhook | Webhook | Receive `order.paid` events |
| Process order | Code | Extract shipping address and order details |
| Ship order | HTTP Request | `POST /v1/orders/{id}/ship` with tracking info |
| Fulfill order | HTTP Request | `POST /v1/orders/{id}/fulfill` when delivered |

### Create the listing

```json
{
  "name": "Handmade Leather Boots",
  "price": 18900,
  "fulfillment": "external",
  "checkout_schema": [
    {
      "key": "size",
      "type": "select",
      "label": "Shoe size",
      "required": true,
      "options": ["EU 38", "EU 39", "EU 40", "EU 41", "EU 42", "EU 43", "EU 44"]
    },
    {
      "key": "shipping_address",
      "type": "address",
      "label": "Shipping address",
      "required": true
    }
  ]
}
```

### Process the order.paid webhook

The `order.paid` event includes `checkout_data` and `shipping_address`:

```json
{
  "type": "order.paid",
  "data": {
    "object": {
      "id": "ord_abc123",
      "status": "paid",
      "buyer_email": "buyer@example.com",
      "checkout_data": {
        "size": "EU 42"
      },
      "shipping_address": {
        "line1": "123 Main St",
        "line2": "Apt 4B",
        "city": "Portland",
        "state": "OR",
        "postal_code": "97201",
        "country": "US"
      },
      "fulfillment_status": "pending"
    }
  }
}
```

Use a **Code** node to extract the shipping details:

```javascript
const order = $json.data.object;
const address = order.shipping_address;

return [{
  json: {
    order_id: order.id,
    buyer_email: order.buyer_email,
    size: order.checkout_data.size,
    ship_to: `${address.line1}, ${address.city}, ${address.state} ${address.postal_code}, ${address.country}`,
  }
}];
```

### Ship the order

After you ship the product, use an **HTTP Request** node:

| Field | Value |
|-------|-------|
| Method | `POST` |
| URL | `https://api.listbee.so/v1/orders/{{ $json.order_id }}/ship` |
| Header | `Authorization: Bearer lb_...` |

**Body:**

```json
{
  "carrier": "USPS",
  "tracking_code": "9400111899223456789012",
  "seller_note": "Ships from Portland. Estimated 3-5 days."
}
```

### Mark as fulfilled

When delivered, use another **HTTP Request** node:

| Field | Value |
|-------|-------|
| Method | `POST` |
| URL | `https://api.listbee.so/v1/orders/{{ $json.order_id }}/fulfill` |
| Header | `Authorization: Bearer lb_...` |

**Body:** `{}`

## Using the ListBee n8n community node

If you prefer a purpose-built node over HTTP Request, install the ListBee community node:

```bash
# In your n8n instance settings → Community nodes → Install
n8n-nodes-listbee
```

The community node provides credential management, typed fields, and auto-completion for all ListBee endpoints. See the [n8n node docs](/n8n) for setup instructions.
