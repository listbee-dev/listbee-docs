Create multiple listings from a JSON catalogue, then paginate through recent orders. Useful for seeding an account from an existing product catalogue.

## Full script

```python
import json
import os
from listbee import ListBee, Deliverable

client = ListBee(api_key=os.environ["LISTBEE_API_KEY"])

# --- Load catalogue ---

with open("catalogue.json") as f:
    catalogue = json.load(f)

# catalogue.json shape:
# [
#   {
#     "name": "Python Automation Handbook",
#     "price": 1900,
#     "description": "Automate repetitive tasks with Python.",
#     "url": "https://example.com/downloads/python-handbook"
#   },
#   ...
# ]

# --- Create + publish ---

created = []

for item in catalogue:
    result = client.listings.create(
        name=item["name"],
        price=item["price"],
        description=item.get("description"),
        fulfillment_mode="static",
        deliverable=Deliverable.url(item["url"]),
    )

    published = client.listings.publish(listing_id=result.listing.id)
    created.append(published)
    print(f"Published: {published.name} — {published.url}")

print(f"\n{len(created)} listings published.")

# --- Poll for recent orders ---

page = client.orders.list(limit=50)
while True:
    for order in page.data:
        print(
            f"{order.id}  {order.status:<12}  "
            f"{order.buyer_email}  "
            f"${order.amount / 100:.2f}"
        )
    if not page.has_more:
        break
    page = client.orders.list(limit=50, cursor=page.cursor)
```

## What happens

**Bulk creation.** Each catalogue entry becomes a `static` listing with a pre-baked URL deliverable. `publish` makes the product page live immediately. When a buyer pays, ListBee delivers the stored URL automatically.

**Cursor pagination.** `list_orders` returns up to 50 orders per page. When `has_more` is true, pass the `cursor` to the next call. The loop exits when `has_more` is false.

## Listing existing listings

```python
page = client.listings.list(limit=100)
while True:
    for listing in page.data:
        print(f"{listing.id}  {listing.status:<10}  {listing.name}")
    if not page.has_more:
        break
    page = client.listings.list(limit=100, cursor=page.cursor)
```

## Variations

**Text deliverables** — pass `Deliverable.text("...")` for license keys or short Markdown content instead of a URL.

**Async generation** — for content generated per-order, use `fulfillment_mode="async"` with an `agent_callback_url`. On `order.paid`, your server generates content and calls `POST /v1/orders/{id}/fulfill`.

**Idempotent runs** — pass an `Idempotency-Key` header on `create_listing` to make re-runs safe. Same key within 24 hours returns the cached response without creating a duplicate.
