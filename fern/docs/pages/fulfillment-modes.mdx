Every listing has a `fulfillment` field: `"managed"` or `"external"`. This determines who delivers the product after payment.

---

## Managed fulfillment

ListBee delivers the product automatically. You attach a deliverable (file, URL, or text) to the listing. After payment, ListBee creates an access grant and delivers it to the buyer by email.

Best for: digital downloads, redirect links, license keys, text content.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Python Async Patterns",
        "price": 1900,
        "content": "https://your-bucket.s3.amazonaws.com/python-async.pdf",
        # fulfillment defaults to "managed" when content is provided
    },
)
listing = resp.json()

print(listing["fulfillment"])       # managed
print(listing["deliverable_type"])  # file
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Python Async Patterns",
  price: 1900,
  content: "https://your-bucket.s3.amazonaws.com/python-async.pdf",
  // fulfillment defaults to "managed" when content is provided
});

console.log(listing.fulfillment);       // managed
console.log(listing.deliverable_type);  // file
```

</CodeGroup>

### What ListBee handles in managed mode

| Step | ListBee does it |
|------|----------------|
| Collect payment | Yes (Stripe Checkout) |
| Create access grant | Yes |
| Deliver to buyer | Yes (email with download link / redirect / inline text) |
| Fire `order.paid` webhook | Yes |
| Fire `order.fulfilled` webhook | Yes |

## External fulfillment

Your app handles delivery. ListBee collects payment, fires an `order.paid` webhook, and your code takes over. No deliverable needed on the listing.

Best for: physical goods, AI-generated content, custom services, anything that requires logic after payment.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Custom Portrait Drawing",
        "price": 4900,
        "fulfillment": "external",
        # no content — your app delivers after payment
    },
)
listing = resp.json()

print(listing["fulfillment"])       # external
print(listing["deliverable_type"])  # None
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Custom Portrait Drawing",
  price: 4900,
  fulfillment: "external",
  // no content — your app delivers after payment
});

console.log(listing.fulfillment);       // external
console.log(listing.deliverable_type);  // null
```

</CodeGroup>

### What you handle in external mode

| Step | Who |
|------|-----|
| Collect payment | ListBee (Stripe Checkout) |
| Receive `order.paid` webhook | Your app |
| Deliver product to buyer | Your app |
| Call `POST /v1/orders/{id}/fulfill` (optional) | Your app |
| Fire `order.fulfilled` webhook | ListBee (after you call fulfill) |

<Warning>
External fulfillment requires a registered webhook. If no webhook is configured, the readiness system shows a `configure_webhook` action — the listing cannot sell until a webhook endpoint is set up to receive `order.paid` events.
</Warning>

---

## Decision tree

```
Do you have a file, URL, or text to deliver automatically?
  → Yes: use managed fulfillment (default when content is provided)
  → No: use external fulfillment

Is delivery handled by your own code or a third party?
  → Yes: use external fulfillment
  → No: use managed fulfillment

Does the product require information from the buyer (address, size, preferences)?
  → Yes: use external fulfillment + checkout_schema
  → No: managed or external — your choice
```

---

## Responsibility matrix

| Responsibility | Managed | External |
|---------------|---------|----------|
| Payment collection | ListBee | ListBee |
| Buyer email collection | ListBee | ListBee |
| Custom checkout fields | Optional (checkout_schema) | Optional (checkout_schema) |
| Digital content delivery | ListBee | Your app |
| Physical goods shipping | N/A | Your app |
| Order status transitions | Automatic (PENDING → PAID → FULFILLED) | PENDING → PAID automatic, PAID → FULFILLED via API |
| Webhook: `order.paid` | Fired | Fired |
| Webhook: `order.fulfilled` | Fired (immediately after paid) | Fired (after you call fulfill or ship) |

---

## Defaults

ListBee infers the fulfillment mode when you don't set it explicitly:

| You provide | Inferred `fulfillment` |
|------------|----------------------|
| `content` (file/URL/text) | `"managed"` |
| No `content` | `"external"` |
| `fulfillment: "external"` + `content` | `"external"` (content ignored for delivery, stored as metadata) |

## Switching modes

Update the fulfillment mode on an existing listing:

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.put(
    "https://api.listbee.so/v1/listings/r7kq2xy9",
    headers={"Authorization": "Bearer lb_..."},
    json={"fulfillment": "external"},
)
listing = resp.json()
print(listing["fulfillment"])  # external
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.update("r7kq2xy9", {
  fulfillment: "external",
});
console.log(listing.fulfillment);  // external
```

</CodeGroup>

```python
# ListBee — fulfillment modes
#
# Every listing has fulfillment: "managed" | "external"
#
# MANAGED (default when content is provided):
#   ListBee delivers automatically — file download, URL redirect, or inline text.

#   ListBee collects payment, fires order.paid webhook, your app delivers.

#   Requires: registered webhook endpoint
#   Events: order.paid fires on payment, order.fulfilled fires after you call fulfill
#
# Create managed listing:
#   POST /v1/listings { name, price, content: "https://..." }
#   → fulfillment: "managed", deliverable_type: "file"
#
# Create external listing:
#   POST /v1/listings { name, price, fulfillment: "external" }
#   → fulfillment: "external", deliverable_type: null
#
# Fulfill an external order:
#   POST /v1/orders/{id}/fulfill  { content?, content_type?, content_url? }
#
# Ship a physical order:
#   POST /v1/orders/{id}/ship  { carrier, tracking_code, seller_note? }
#
# Auth: Authorization: Bearer lb_...
# Docs: https://docs.listbee.so/fulfillment-modes
```
