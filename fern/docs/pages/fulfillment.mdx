Fulfillment is what happens after a buyer pays. The behavior is determined by the listing's `content_type`, set at creation and locked on publish. There are three modes: ListBee delivers pre-attached content automatically (`static`), your agent pushes generated content (`generated`), or your external system takes over entirely (`webhook`).

## How it works

When Stripe confirms payment, ListBee inspects the order's `content_type` and takes the appropriate action:

- **static** — ListBee immediately creates an access grant from the listing's pre-attached deliverables, emails the buyer a download link, and marks the order `fulfilled`. No action required from you.
- **generated** — the order transitions to `processing` and ListBee fires `order.paid`. Your agent receives the event, generates the content, then calls `POST /v1/orders/{id}/fulfill` to push it. ListBee delivers it and marks the order `fulfilled`.
- **webhook** — ListBee fires `order.paid` and marks the order `handed_off`. Your system is fully responsible for delivery. ListBee does not deliver anything.

`order.fulfilled` fires for `static` and `generated` orders. `order.paid` fires for all three.

## Choosing a content type

| | `static` | `generated` | `webhook` |
|---|---|---|---|
| Who delivers | ListBee | ListBee (after agent pushes) | Your system |
| Deliverables on listing | Required | Not allowed | Not allowed |
| Webhook required | No | Yes (`order.paid`) | Yes (`order.paid`) |
| Order terminal state | `fulfilled` | `fulfilled` | `handed_off` |
| Use when | Fixed files, URLs, text | AI-generated content, per-order reports | Physical goods, services, SaaS access |

## Static delivery

Attach up to 3 deliverables to the listing before publishing. On payment, ListBee delivers them automatically.

<CodeBlocks>
  <CodeBlock title="Python">
```python
from listbee import ListBee

client = ListBee(api_key="lb_...")

# 1. Create a static listing
listing = client.listings.create(
    name="Python Crash Course",
    price=2900,
    content_type="static",
)

# 2. Attach content (file, url, or text — up to 3)
client.listings.set_deliverables(
    listing_id=listing.id,
    deliverables=[
        {"type": "url", "value": "https://cdn.example.com/python-course.pdf"},
    ],
)

# 3. Publish — buyers can now pay and receive the PDF automatically
client.listings.publish(listing_id=listing.id)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
# Create listing
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "Python Crash Course", "price": 2900, "content_type": "static"}'

# Attach deliverable
curl -X PUT https://api.listbee.so/v1/listings/lst_7kQ2xY9mN3pR5tW1vB8a/deliverables \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"deliverables": [{"type": "url", "value": "https://cdn.example.com/python-course.pdf"}]}'

# Publish
curl -X POST https://api.listbee.so/v1/listings/lst_7kQ2xY9mN3pR5tW1vB8a/publish \
  -H "Authorization: Bearer lb_..."
```
  </CodeBlock>
</CodeBlocks>

## Generated delivery

Your agent generates content per order in response to `order.paid`, then calls `fulfill` to push it back.

<CodeBlocks>
  <CodeBlock title="Python">
```python
# 1. Create a generated listing (no deliverables needed)
listing = client.listings.create(
    name="Personalized Investment Report",
    price=4900,
    content_type="generated",
)

# Publish (requires an active order.paid webhook first — see readiness)
client.listings.publish(listing_id=listing.id)

# 2. In your webhook handler — called when order.paid fires:
def handle_order_paid(order_id: str, checkout_data: dict):
    # Generate content for this buyer
    report = generate_report(checkout_data)

    # Push it to ListBee — buyer receives email with download link
    client.orders.fulfill(
        order_id=order_id,
        deliverables=[
            {"type": "text", "value": report},
        ],
    )
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
# Push generated content to an order
curl -X POST https://api.listbee.so/v1/orders/ord_9xM4kP7nR2qT5wY1/fulfill \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "deliverables": [
      {"type": "text", "value": "Your personalized investment report: ..."}
    ]
  }'
```
  </CodeBlock>
</CodeBlocks>

## Webhook (external) delivery

ListBee fires `order.paid` and hands off. Your system handles all delivery.

<CodeBlocks>
  <CodeBlock title="Python">
```python
# 1. Create a webhook listing
listing = client.listings.create(
    name="Premium Membership",
    price=9900,
    content_type="webhook",
)

client.listings.publish(listing_id=listing.id)

# 2. Your endpoint receives order.paid and handles delivery:
#    - Provision membership access
#    - Send welcome email
#    - Any delivery mechanism you control
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "Premium Membership", "price": 9900, "content_type": "webhook"}'
```
  </CodeBlock>
</CodeBlocks>

## Deliverable types

A deliverable is what the buyer receives after payment. Each has a `type` and either a `token` (for files) or a `value` (for URLs and text). Up to 3 deliverables per listing or per `fulfill` call. Mixed types allowed.

| Type | Field | What the buyer gets | Limits |
|------|-------|-------------------|--------|
| `file` | `token` from `POST /v1/files` | Signed download URL | 100 MB max, 48h expiry, 10 downloads |
| `url` | `value` — any URL | Redirect to that URL | No size limit |
| `text` | `value` — literal string | Text rendered on download page | 10,000 chars |

### File deliverable

Upload first, then reference the token. Two steps:

<CodeBlocks>
  <CodeBlock title="Python">
```python
# Step 1: Upload the file
file = client.files.upload(open("ebook.pdf", "rb"))
print(file.id)  # file_7kQ2xY9mN3pR5tW1vB8a01

# Step 2: Attach to listing
client.listings.set_deliverables(
    listing_id="lst_abc123",
    deliverables=[{"type": "file", "token": file.id}],
)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
# Step 1: Upload
curl -X POST https://api.listbee.so/v1/files \
  -H "Authorization: Bearer lb_..." \
  -F "file=@ebook.pdf"
# Returns: {"object": "file", "id": "file_7kQ2xY9mN3pR5tW1vB8a01", ...}

# Step 2: Attach
curl -X PUT https://api.listbee.so/v1/listings/lst_abc123/deliverables \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"deliverables": [{"type": "file", "token": "file_7kQ2xY9mN3pR5tW1vB8a01"}]}'
```
  </CodeBlock>
</CodeBlocks>

After payment, the buyer gets an access grant with a signed download URL. The grant expires after 48 hours or 10 downloads, whichever comes first.

### URL deliverable

Direct redirect — the buyer clicks the download link and is sent to your URL. No file hosting needed.

```json
{"type": "url", "value": "https://cdn.example.com/guide.pdf"}
```

Use this for content already hosted elsewhere (S3, Google Drive shared links, Notion pages, Gumroad-style external downloads).

### Text deliverable

Literal text rendered on the download page. Good for license keys, passwords, short instructions, or generated content.

```json
{"type": "text", "value": "Your license key: ABC-123-XYZ\n\nActivate at https://example.com/activate"}
```

### Mixing types

A single listing can combine all three:

```json
{
  "deliverables": [
    {"type": "file", "token": "file_7kQ2xY9mN3pR5tW1vB8a01"},
    {"type": "url", "value": "https://community.example.com/invite/abc"},
    {"type": "text", "value": "Welcome! Your login: user@example.com / temp-pass-123"}
  ]
}
```

The buyer sees all three on their download page after payment.
