Fulfillment is what happens after a buyer pays. The behavior is derived from whether the listing has deliverables attached — no explicit mode to configure.

## How it works

When Stripe confirms payment, ListBee inspects the order and takes the appropriate action:

- **Has deliverables** — ListBee immediately creates an access grant, emails the buyer a download link, and marks the order `fulfilled`. No action required from you.
- **No deliverables** — ListBee fires `order.paid` to your configured webhook. Your system handles delivery. Optionally, your agent can call `POST /v1/orders/{id}/fulfill` to push content back to ListBee for delivery.

`order.fulfilled` fires when an order reaches the `fulfilled` state. `order.paid` fires for every order.

## The `fulfillment_url` callback

Set `fulfillment_url` on a listing to receive a POST request from ListBee when each order is paid. This is a per-listing callback — separate from your account-level webhooks.

The payload includes enriched metadata so your agent can act without any lookups:

```json
{
  "order_id": "ord_9xM4kP7nR2qT5wY1",
  "listing_id": "lst_7kQ2xY9mN3pR5tW1vB8a",
  "buyer_email": "buyer@example.com",
  "amount": 4900,
  "has_deliverables": false,
  "listing_metadata": {"product_tier": "pro", "template_id": "rpt-42"},
  "checkout_data": {"company": "Acme Corp", "use_case": "quarterly review"},
  "fulfill_url": "https://api.listbee.so/v1/orders/ord_9xM4kP7nR2qT5wY1/fulfill"
}
```

`fulfill_url` is the endpoint your agent calls to push generated content back. `fulfillment_url` is where ListBee calls your agent — they are distinct.

<CodeBlocks>
  <CodeBlock title="Python">
```python
from listbee import ListBee

client = ListBee(api_key="lb_...")

listing = client.listings.create(
    name="Personalized Investment Report",
    price=4900,
    fulfillment_url="https://your-agent.example.com/hooks/listbee",
    metadata={"product_tier": "pro", "template_id": "rpt-42"},
)

client.listings.publish(listing_id=listing.id)
```
  </CodeBlock>


## The `/fulfill` endpoint

Call `POST /v1/orders/{id}/fulfill` to push deliverables to ListBee for any `paid` order. ListBee creates an access grant, emails the buyer, and marks the order `fulfilled`.

Works on any PAID order regardless of whether the listing has deliverables. Idempotent — calling it on an already-`fulfilled` order is a no-op.

<CodeBlocks>
  <CodeBlock title="Python">
```python
# In your webhook or fulfillment_url handler:
client.orders.fulfill(
    order_id="ord_9xM4kP7nR2qT5wY1",
    deliverables=[
        {"type": "text", "value": "Your personalized investment report: ..."},
    ],
)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/orders/ord_9xM4kP7nR2qT5wY1/fulfill \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "deliverables": [
      {"type": "text", "value": "Your personalized investment report: ..."}
    ]
  }'
```
  </CodeBlock>
</CodeBlocks>

## Deliverables

A deliverable is the content a buyer receives — a file, URL, or text. Attach deliverables to a listing for instant delivery, or push them via `/fulfill` for generated content. Up to 3 per listing, mixed types allowed.

For types, hosting, file uploads, security, and limits — see [Deliverables](/docs/deliverables).

## The stateless agent pattern

`fulfillment_url` and `metadata` together enable a stateless agent: embed everything the agent needs to generate content at the time you create the listing. When `order.paid` fires, the enriched payload carries it all — no database lookups, no state management.

```python
# At listing creation time — embed generation parameters in metadata
listing = client.listings.create(
    name="Custom Market Analysis",
    price=9900,
    fulfillment_url="https://your-agent.example.com/generate",
    metadata={
        "report_type": "market_analysis",
        "data_sources": ["bloomberg", "sec_filings"],
        "output_format": "pdf",
    },
)

# At fulfillment time — your agent receives everything it needs in the payload:
# {
#   "listing_metadata": {"report_type": "market_analysis", ...},
#   "checkout_data": {"ticker": "AAPL", "period": "Q1 2026"},
#   "fulfill_url": "https://api.listbee.so/v1/orders/ord_.../fulfill"
# }
#
# Generate → POST to fulfill_url → done. No state stored on your side.
```
