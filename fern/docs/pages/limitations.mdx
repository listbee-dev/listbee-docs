## Scope

**No marketplace.** ListBee is a single-seller API. Each account has one brand and one Stripe Connect connection. There is no multi-vendor marketplace, no seller discovery, and no cross-account order routing. Agents operating on behalf of multiple brands use multiple accounts.

**No physical shipping.** ListBee does not calculate shipping rates, generate labels, or integrate with fulfillment providers. You can collect a shipping address at checkout using `checkout_schema`, but ListBee only passes that data to you in the `order.paid` payload — your app handles fulfillment.

**USD only.** All prices and charges are in US dollars. There is no multi-currency support.

**No subscriptions or recurring billing.** ListBee processes one-time payments only. For recurring revenue, use Stripe Billing directly.

**No built-in analytics dashboard.** Order and listing data is accessible via the API — build your own reporting, or pipe `GET /v1/events` to a data warehouse.

## No file hosting

ListBee does not store files. Deliverables are either:

- a URL you host and redirect buyers to, or
- a block of Markdown text ListBee renders inline.

If you need to sell a file, host it somewhere (S3, CDN, Google Drive, signed-URL service) and pass that URL as a `url` deliverable.

## Unlock URLs

Every order has a permanent `unlock_url` — no expiry, no view counter, no per-buyer revocation. Treat it like a bearer token. If you need stricter access, host the content at a URL that enforces it and deliver that URL.

## Rate limits

API requests are rate-limited per API key. See [Rate limits](/rate-limits).

## Webhooks

The `agent_callback_url` on a listing has a 30-second delivery timeout. Failures retry on a 7-attempt schedule spanning ~24 hours. After that, use `POST /v1/orders/{id}/redeliver` or poll `GET /v1/events` to reconcile. See [Fulfillment](/fulfillment).

There is no event-ordering guarantee across listings. If ordering matters within a listing, process callbacks serially in your receiver.

## Stripe requirements

Sellers must complete Stripe Connect onboarding before a listing can sell. Listings can be created and published earlier — the product page renders with a disabled buy button until the account is operational. See [Readiness](/readiness).

ListBee collects a platform fee on each sale via Stripe Connect. The fee rate is shown on the account object (`fee_rate`).
