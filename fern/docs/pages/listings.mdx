A listing is a sellable item. It has a price, a fulfillment mode, and a hosted product page buyers visit to pay. Creating a listing returns a shareable URL and a one-time `signing_secret` for webhook signature verification.

## Lifecycle

```
created  →  DRAFT  ⇄  PUBLISHED  →  ARCHIVED
```

- **`draft`** — editable, not visible to buyers
- **`published`** — live on the product page URL, orders can be placed
- **`archived`** — terminal, no longer visible, cannot be published again

Transitions: `/publish`, `/unpublish` (back to draft — agent's "pause"), `/archive` (terminal).

Whether a published listing is actually buyable also depends on account readiness (Stripe Connect, charges enabled). See [Readiness](/readiness).

## Fulfillment modes

Set at creation, mutable via PATCH:

- **`static`** — bake the deliverable into the listing. ListBee delivers it at `order.paid` and transitions the order to `fulfilled`.
- **`async`** — no deliverable on the listing. After each order, the agent pushes content via `POST /v1/orders/{id}/fulfill`.

## Creating a listing

Required fields: `name` and `price`. Everything else is optional.

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import ListBee, Deliverable

client = ListBee(api_key="lb_...")

created = client.listings.create(
    name="SEO Playbook 2026",
    price=1900,                             # cents
    fulfillment_mode="static",
    deliverable=Deliverable.url("https://example.com/seo-playbook.pdf"),
    description="A comprehensive guide to modern SEO techniques.",
    tagline="Updated for 2026 algorithm changes",
    agent_callback_url="https://your-agent.example.com/hooks/listbee",
    metadata={"campaign": "spring-launch"},
)

listing = created.listing
signing_secret = created.signing_secret     # shown once — store immediately
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "SEO Playbook 2026",
    "price": 1900,
    "fulfillment_mode": "static",
    "deliverable": {"type": "url", "value": "https://example.com/seo-playbook.pdf"},
    "description": "A comprehensive guide to modern SEO techniques.",
    "tagline": "Updated for 2026 algorithm changes",
    "agent_callback_url": "https://your-agent.example.com/hooks/listbee",
    "metadata": {"campaign": "spring-launch"}
  }'
```
</CodeBlock>
</CodeBlocks>

The response wraps the listing and the one-time signing secret:

```json
{
  "object": "listing_with_secret",
  "listing": { "object": "listing", "id": "lst_...", "status": "draft", ... },
  "signing_secret": "Xk9mN2pR5tW1vB8aQ4nL7xY0zJ3eF6cH"
}
```

Subsequent reads only expose `signing_secret_preview` (last 4 chars).

## Signing secret

Each listing has a per-listing HMAC key used to sign webhook payloads to `agent_callback_url`. See [Fulfillment](/fulfillment#verifying-signatures) for verification.

Supply your own on create:

```python
client.listings.create(
    name="...",
    price=1900,
    signing_secret="my-pre-shared-secret-agents-already-know",
)
```

Or rotate via PATCH:

```python
# Pass a new value, or null to auto-generate
client.listings.update(listing_id=listing.id, signing_secret=None)
# response: RotateSigningSecretResponse with the full new secret (shown once)
```

Rotation invalidates the old secret immediately — there's no grace window.

## Agent callback URL

Set `agent_callback_url` to receive `order.paid`, `order.fulfilled`, `order.refunded`, and `order.disputed` events for this listing, signed with `signing_secret`.

```python
client.listings.update(
    listing_id=listing.id,
    agent_callback_url="https://your-agent.example.com/hooks/listbee",
)
```

Leave it null to rely on polling `/v1/orders` or `/v1/events` instead. See [Events](/events) and [Fulfillment](/fulfillment).

## Metadata

`metadata` is a free-form dict that rides with the listing into order snapshots and webhook payloads. Use it to carry generation parameters or correlation keys your agent needs, without shadow state.

Stripe-aligned limits: max 50 keys, keys ≤ 40 chars, values must be strings ≤ 500 chars, no nested objects.

```python
client.listings.create(
    name="Custom Market Analysis",
    price=9900,
    fulfillment_mode="async",
    agent_callback_url="https://your-agent.example.com/generate",
    metadata={
        "report_type": "market_analysis",
        "data_sources": "bloomberg,sec_filings",
        "output_format": "pdf",
    },
)
```

## State transitions

<CodeBlocks>
<CodeBlock title="Python">
```python
client.listings.publish(listing_id=listing.id)     # draft → published
client.listings.unpublish(listing_id=listing.id)   # published → draft
client.listings.archive(listing_id=listing.id)     # → archived (terminal)
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/listings/lst_.../publish    -H "Authorization: Bearer lb_..."
curl -X POST https://api.listbee.so/v1/listings/lst_.../unpublish  -H "Authorization: Bearer lb_..."
curl -X POST https://api.listbee.so/v1/listings/lst_.../archive    -H "Authorization: Bearer lb_..."
```
</CodeBlock>
</CodeBlocks>

`publish` returns `409` with a readiness payload if the listing isn't sellable (e.g. `static` mode with no deliverable). Check `readiness.actions` to see what's blocking.

## Updating

Partial updates — only provided fields change.

```python
client.listings.update(
    listing_id=listing.id,
    price=2500,
    tagline="Now with bonus templates",
    deliverable=Deliverable.url("https://example.com/seo-playbook-v2.pdf"),
)
```

All fields including `fulfillment_mode`, `deliverable`, `agent_callback_url`, `metadata` are mutable at any time.

## Listing all listings

```python
result = client.listings.list()
for listing in result.data:
    print(listing.id, listing.name, listing.status)

if result.has_more:
    next_page = client.listings.list(cursor=result.cursor)
```

## Key fields

| Field                   | Type    | Description |
|-------------------------|---------|-------------|
| `id`                    | string  | Unique listing ID (`lst_`) |
| `status`                | string  | `draft`, `published`, or `archived` |
| `fulfillment_mode`      | string  | `static` or `async` |
| `deliverable`           | object  | Pre-baked deliverable (static mode only), `null` otherwise |
| `agent_callback_url`    | string  | Per-listing webhook URL; `null` to disable |
| `signing_secret_preview`| string  | Last 4 chars of the signing secret (full value only at create/rotate) |
| `metadata`              | object  | Forwarded in webhook payloads |
| `price`                 | integer | Cents — `1900` = $19.00 |
| `checkout_schema`       | array   | Custom fields collected at checkout |
| `short_code`            | string  | Identifier in the product page URL |
| `url`                   | string  | Product page URL |
| `readiness`             | object  | `sellable` flag + `actions` when not sellable |
