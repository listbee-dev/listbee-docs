A listing is a sellable item. It has a price and a hosted product page buyers visit to pay. Creating a listing returns a URL you share with buyers. When a buyer pays, ListBee delivers content automatically if the listing has deliverables — otherwise it fires a webhook for your system to handle.

## How it works

Listings start in `draft` status. In draft, all fields are editable and the listing is not yet visible to buyers. When you're ready, call `publish` — the listing goes live, the product page becomes accessible, and orders can be placed.

The product page at `url` (e.g. `https://buy.listbee.so/aB3xK9m`) shows the listing name, description, price, and cover image. Buyers click the buy button and pay via Stripe Checkout. What happens after payment depends on whether the listing has deliverables attached — see [Fulfillment](/docs/fulfillment).

Each listing must pass a readiness check before it can be published. `readiness.sellable` indicates whether a buyer can currently complete a purchase. If `false`, `readiness.actions` lists what's needed. See [Readiness](/docs/readiness).

## Creating a listing

The two required fields are `name` and `price` (in cents).

<CodeBlocks>
  <CodeBlock title="Python">
```python
from listbee import ListBee

client = ListBee(api_key="lb_...")

listing = client.listings.create(
    name="SEO Playbook 2026",
    price=1900,             # $19.00
    description="A comprehensive guide to modern SEO techniques.",
    tagline="Updated for 2026 algorithm changes",
)

print(listing.id)              # lst_7kQ2xY9mN3pR5tW1vB8a
print(listing.status)          # draft
print(listing.short_code)      # aB3xK9m
print(listing.url)             # https://buy.listbee.so/aB3xK9m
print(listing.has_deliverables) # False
```
  </CodeBlock>
  <CodeBlock title="TypeScript">
```typescript
import { ListBee } from 'listbee';

const client = new ListBee({ apiKey: 'lb_...' });

const listing = await client.listings.create({
  name: 'SEO Playbook 2026',
  price: 1900,
  description: 'A comprehensive guide to modern SEO techniques.',
  tagline: 'Updated for 2026 algorithm changes',
});

console.log(listing.id);              // lst_7kQ2xY9mN3pR5tW1vB8a
console.log(listing.hasDeliverables); // false
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "SEO Playbook 2026",
    "price": 1900,
    "description": "A comprehensive guide to modern SEO techniques.",
    "tagline": "Updated for 2026 algorithm changes"
  }'
```
  </CodeBlock>
</CodeBlocks>

To use a per-listing callback for agent-driven fulfillment, pass `fulfillment_url`. ListBee POSTs an enriched payload to this URL on each purchase. You can change `fulfillment_url` anytime, including after publishing.

<CodeBlocks>
  <CodeBlock title="Python">
```python
listing = client.listings.create(
    name="Personalized Market Report",
    price=4900,
    fulfillment_url="https://your-agent.example.com/hooks/listbee",
    metadata={"report_type": "market_analysis"},
)
```
  </CodeBlock>
  <CodeBlock title="TypeScript">
```typescript
const listing = await client.listings.create({
  name: 'Personalized Market Report',
  price: 4900,
  fulfillmentUrl: 'https://your-agent.example.com/hooks/listbee',
  metadata: { reportType: 'market_analysis' },
});
```
  </CodeBlock>

## Attaching deliverables

Attach content that buyers receive after payment. Any draft listing can have deliverables. If deliverables are attached, ListBee auto-delivers on payment.

<CodeBlocks>
  <CodeBlock title="Python">
```python
from listbee import Deliverable

client.listings.set_deliverables(
    listing_id="lst_7kQ2xY9mN3pR5tW1vB8a",
    deliverables=[Deliverable.file("ebook.pdf")],
)
```
  </CodeBlock>
  <CodeBlock title="TypeScript">
```typescript
import { Deliverable } from 'listbee';

await client.listings.setDeliverables('lst_7kQ2xY9mN3pR5tW1vB8a', {
  deliverables: [Deliverable.file('ebook.pdf')],
});
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X PUT https://api.listbee.so/v1/listings/lst_7kQ2xY9mN3pR5tW1vB8a/deliverables \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"deliverables": [{"type": "file", "token": "file_7kQ2xY9mN3pR5tW1vB8a01"}]}'
```
  </CodeBlock>
</CodeBlocks>

Three types: `file`, `url`, and `text`. The `Deliverable` builder handles file uploads automatically. For the full reference on types, hosting, security, and limits — see [Deliverables](/docs/deliverables).

## Publishing

Once Stripe is connected — and the listing has deliverables or a fulfillment path — publish the listing:

<CodeBlocks>
  <CodeBlock title="Python">
```python
published = client.listings.publish(listing_id="lst_7kQ2xY9mN3pR5tW1vB8a")

print(published.status)                # published
print(published.readiness.sellable)    # True
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/listings/lst_7kQ2xY9mN3pR5tW1vB8a/publish \
  -H "Authorization: Bearer lb_..."
```
  </CodeBlock>
</CodeBlocks>

Publishing fails with `409` if `readiness.sellable` is `false`. Check `readiness.actions` to see what's blocking. A listing without deliverables can still be published — the readiness check accounts for webhook-based fulfillment paths.

## Updating a listing

Partial updates — only provided fields change. All fields, including `fulfillment_url`, can be changed at any time.

## Key fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique listing ID (`lst_` prefixed) |
| `name` | string | Product name shown on the product page |
| `short_code` | string | 7-character base62 identifier used in the product page URL |
| `price` | integer | Price in cents — `1900` = $19.00 |
| `status` | string | `draft` or `published` |
| `has_deliverables` | boolean | Whether this listing has deliverables attached |
| `fulfillment_url` | string | Per-listing callback URL; ListBee POSTs enriched payload on purchase |
| `url` | string | Product page URL — share this with buyers |
| `embed_url` | string | Embeddable product page URL for iframes |
| `deliverables` | array | Content attached to this listing |
| `checkout_schema` | array | Custom fields collected from the buyer at checkout |
| `stock` | integer | `null` = unlimited; `0` = out of stock; positive = remaining units |
| `compare_at_price` | integer | Strikethrough price in cents |
| `metadata` | object | Arbitrary key-value pairs; included in `fulfillment_url` callbacks |
| `readiness` | object | Whether the listing is sellable and what actions are needed |
| `readiness.sellable` | boolean | `true` when buyers can complete a purchase |

## State transitions

```
draft → published   (via POST /v1/listings/{id}/publish)
```

Published listings cannot be unpublished. Delete and recreate if needed.
