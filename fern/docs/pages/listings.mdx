A listing is a priced product with a hosted product page. Create one with a name, price, and optional deliverable â€” ListBee generates a URL you can share with buyers.

Each listing has a `readiness.sellable` flag. When `true`, buyers can purchase immediately. When `false`, `readiness.actions` tells you exactly what to fix. See the [Readiness system](/readiness) for full details.

---

## Create a listing

`POST /v1/listings` requires `name` and `price` (in cents). Currency is determined by your account settings.

### Managed fulfillment (digital content)

Pass `deliverable` â€” a URL pointing to a file, a redirect URL, or a plain-text string. ListBee detects the deliverable type and handles delivery automatically.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Python Async Patterns",
        "price": 1900,      # $19.00 in cents
        "deliverable": "https://your-bucket.s3.amazonaws.com/python-async-patterns.pdf",
    },
)
listing = resp.json()

print(listing["url"])                    # https://buy.listbee.so/r7kq2xy9
print(listing["fulfillment"])            # managed
print(listing["deliverable_type"])       # file
print(listing["readiness"]["sellable"])  # True
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Python Async Patterns",
  price: 1900,     // $19.00 in cents
  deliverable: "https://your-bucket.s3.amazonaws.com/python-async-patterns.pdf",
});

console.log(listing.url);                   // https://buy.listbee.so/r7kq2xy9
console.log(listing.fulfillment);           // managed
console.log(listing.deliverable_type);      // file
console.log(listing.readiness.sellable);    // true
```

</CodeGroup>

### External fulfillment (your app delivers)

Set `fulfillment: "external"` â€” no deliverable needed. Add a `checkout_schema` to collect buyer information (address, preferences, etc.).

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Handmade Leather Boots",
        "price": 18900,
        "fulfillment": "external",
        "checkout_schema": [
            {"key": "size", "type": "select", "label": "Shoe size", "required": True, "options": ["EU 38", "EU 39", "EU 40", "EU 41", "EU 42"]},
            {"key": "shipping_address", "type": "address", "label": "Shipping address", "required": True},
        ],
    },
)
listing = resp.json()

print(listing["fulfillment"])            # external
print(listing["deliverable_type"])       # None
print(listing["readiness"]["sellable"])  # True (if webhook configured)
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.create({
  name: "Handmade Leather Boots",
  price: 18900,
  fulfillment: "external",
  checkoutSchema: [
    { key: "size", type: "select", label: "Shoe size", required: true, options: ["EU 38", "EU 39", "EU 40", "EU 41", "EU 42"] },
    { key: "shipping_address", type: "address", label: "Shipping address", required: true },
  ],
});

console.log(listing.fulfillment);           // external
console.log(listing.deliverable_type);      // null
console.log(listing.readiness.sellable);    // true (if webhook configured)
```

</CodeGroup>

For multi-store accounts, pass `store_id` to assign the listing to a specific store. Single-store accounts can omit it.

## Deliverable types

ListBee detects deliverable type from the `deliverable` field:

| Type | What you provide | What happens (managed) |
|------|-----------------|----------------------|
| `FILE` | A URL ending in a file extension (`.pdf`, `.zip`, etc.) | File fetched and hosted on CDN. Buyers receive a signed URL. |
| `URL` | Any other URL | After payment, buyers are redirected to the URL. |
| `TEXT` | A plain string (no `http://` prefix) | Inline content delivered in the order confirmation. |
| `null` | No deliverable (external fulfillment) | Your app handles delivery. |

See [Deliverables](/deliverables) for edge cases and the full detection logic.

## Update a listing

`PUT /v1/listings/{slug}` accepts any subset of fields. Only the fields you send are changed.

The deliverable is immutable after creation â€” to change what buyers receive, delete the listing and create a new one. Fulfillment mode and checkout schema can be updated.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.put(
    "https://api.listbee.so/v1/listings/r7kq2xy9",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "price": 2500,   # raise price to $25.00
        "tagline": "Production-ready async patterns for Python 3.10+",
    },
)
listing = resp.json()
print(listing["price"])    # 2500
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const listing = await client.listings.update("r7kq2xy9", {
  price: 2500,   // raise price to $25.00
  tagline: "Production-ready async patterns for Python 3.10+",
});
console.log(listing.price);    // 2500
```

</CodeGroup>

## Delete a listing

`DELETE /v1/listings/{slug}` removes the listing and its product page. If the deliverable type was `FILE`, the file is also removed from the CDN.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.delete(
    "https://api.listbee.so/v1/listings/r7kq2xy9",
    headers={"Authorization": "Bearer lb_..."},
)
# 204 No Content on success
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

await client.listings.delete("r7kq2xy9");
// Returns void on success
```

</CodeGroup>

## Key fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Listing title shown on the product page. |
| `price` | integer | Price in smallest currency unit (cents for USD). |
| `fulfillment` | string | `"managed"` or `"external"`. |
| `deliverable_type` | string \| null | `"file"`, `"url"`, `"text"`, or `null`. |
| `checkout_schema` | array \| null | Custom fields collected at checkout. Max 10. |
| `description` | string | Full description. Markdown supported. |
| `tagline` | string | Short one-line pitch shown under the title. |
| `highlights` | string[] | Bullet points displayed prominently on the page. |
| `cta` | string | Buy button label. Defaults to `"Buy now"`. |
| `cover_url` | string | URL of the cover image shown on the product page. |
| `metadata` | object | Key-value store for your own data. Up to 50 keys. |
| `badges` | string[] | Small labels displayed on the listing (e.g. `"New"`, `"Bestseller"`). |
| `rating` | number | Display rating (0-5). Does not affect functionality. |
| `reviews` | integer | Display review count. Does not affect functionality. |
| `faqs` | object[] | FAQ items with `question` and `answer` fields. |

## Slug-based lookup

Listings are fetched by slug, not ID. The slug is derived from the listing name at creation time and cannot be changed.

```
GET  /v1/listings/{slug}
PUT  /v1/listings/{slug}
DELETE /v1/listings/{slug}
```
