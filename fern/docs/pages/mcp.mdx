ListBee ships an MCP server (`listbee-mcp`) that exposes your commerce API as tools any MCP-compatible agent can call. Agents can create listings, check orders, configure webhooks, and trigger fulfillment ‚Äî without writing API integration code.

## Install

The server runs via `npx`. No global install required.

```bash
npx listbee-mcp
```

Requires Node.js 20 or later.

## Configuration

Set your API key as an environment variable before starting the server:

```bash
export LISTBEE_API_KEY=lb_...
npx listbee-mcp
```

### Claude Desktop

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "listbee": {
      "command": "npx",
      "args": ["listbee-mcp"],
      "env": {
        "LISTBEE_API_KEY": "lb_..."
      }
    }
  }
}
```

### Cursor / Cline / other stdio clients

Same pattern ‚Äî pass the command `npx listbee-mcp` and set `LISTBEE_API_KEY` in the process environment.

## Transports

The package ships two transports from the same codebase:

| Transport | How to use | Typical clients |
|-----------|-----------|-----------------|
| **stdio** | `npx listbee-mcp` | Claude Desktop, Cursor, Cline, Claude Code |
| **HTTP** | `mcp.listbee.so` | ChatGPT Apps, Claude API Connector, remote agents |

For the HTTP transport, point your client at `https://mcp.listbee.so`. Requests are authenticated with your `LISTBEE_API_KEY` in the `Authorization` header.

MCP discovery is available at [listbee.so/.well-known/mcp.json](https://listbee.so/.well-known/mcp.json).

## Available tools

The server exposes 27 tools covering the full API surface. Tools are grouped by resource:

**Listings**
- `create_listing` ‚Äî create a draft listing
- `get_listing` ‚Äî retrieve a listing by ID
- `list_listings` ‚Äî paginate your listings
- `update_listing` ‚Äî update name, price, description, checkout schema
- `delete_listing` ‚Äî delete a draft listing
- `add_deliverable` ‚Äî attach a deliverable (file, URL, or text)
- `set_deliverables` ‚Äî replace all deliverables at once
- `remove_deliverable` ‚Äî remove a single deliverable
- `remove_deliverables` ‚Äî remove all deliverables
- `publish_listing` ‚Äî make a listing live

**Orders**
- `list_orders` ‚Äî paginate orders, filter by status or listing
- `get_order` ‚Äî retrieve an order and its buyer data
- `fulfill_order` ‚Äî push generated content to an order
- `refund_order` ‚Äî issue a full refund

**Files**
- `upload_file` ‚Äî upload a file for use as a deliverable

**Webhooks**
- `create_webhook` ‚Äî register a webhook endpoint
- `list_webhooks` ‚Äî list configured webhooks
- `update_webhook` ‚Äî change URL, secret, or event filters
- `delete_webhook` ‚Äî remove a webhook
- `list_webhook_events` ‚Äî inspect delivery history
- `retry_webhook_event` ‚Äî re-deliver a failed event
- `test_webhook` ‚Äî send a test payload

**Account**
- `get_account` ‚Äî retrieve account details and readiness
- `update_account` ‚Äî update display name, bio, avatar
- `start_stripe_connect` ‚Äî begin Stripe Connect onboarding
- `list_customers` ‚Äî paginate buyers
- `list_plans` ‚Äî retrieve available billing plans

## Readiness-aware responses

Every listing and account response includes a `readiness` object with an `actions` array. Each action has a `code`, a `kind` (`api` or `human`), a human-readable `message`, and a `resolve` block with the exact method and endpoint to call next.

Agents should read `readiness.next` to know what to do after each step ‚Äî the API walks agents through the setup sequence automatically.
