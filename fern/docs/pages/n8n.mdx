The `n8n-nodes-listbee` package adds ListBee as a native node in n8n. Create listings, retrieve orders, fulfill async content, and trigger workflows on incoming order events — all from the n8n workflow editor.

## Install

In your n8n instance, go to **Settings → Community nodes** and install:

```
n8n-nodes-listbee
```

Or install directly if you run n8n via Docker or npm:

```bash
npm install n8n-nodes-listbee
```

Restart n8n. The **ListBee** and **ListBee Trigger** nodes appear in the node picker.

## Configure credentials

1. Add a **ListBee** node to any workflow.
2. Click **Credential for ListBee API** → **Create new**.
3. Paste an API key (starts with `lb_`). Get one from [console.listbee.so](https://console.listbee.so) or via the [bootstrap flow](/authentication).
4. Click **Save**.

The credential is shared across all ListBee nodes in the instance.

## Workflow: create a listing and react to orders

One listing with an `agent_callback_url` pointed at n8n's trigger, plus downstream steps to act on each order.

### 1. Add a ListBee Trigger node

The trigger exposes a webhook URL at `https://your-n8n.example.com/webhook/<id>`. Select the events to listen for (`order.paid`, `order.fulfilled`, `order.refunded`, `order.disputed`). Copy the URL — you'll wire it into a listing next.

### 2. Create a listing pointing at the trigger URL

Add a **ListBee** node:

- **Resource:** Listing
- **Operation:** Create
- **Name:** `My Ebook`
- **Price:** `1900` (cents)
- **Fulfillment Mode:** `static` or `async`
- **Agent Callback URL:** the trigger URL from step 1

<CodeBlocks>
<CodeBlock title="curl equivalent">
```bash
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Ebook",
    "price": 1900,
    "fulfillment_mode": "static",
    "deliverable": {"type": "url", "value": "https://example.com/ebook.pdf"},
    "agent_callback_url": "https://your-n8n.example.com/webhook/<id>"
  }'
```
</CodeBlock>
</CodeBlocks>

The response returns the `signing_secret` once. Save it — the trigger node uses it to verify the HMAC-SHA256 signature on every incoming event.

### 3. Act on the order downstream

When a buyer pays, the trigger fires with the full event payload. Wire any downstream node — Slack, Google Sheets, HTTP request, LLM, the ListBee node set to `fulfill_order` — to process each order.

```json
{
  "id": "evt_abc123",
  "type": "order.paid",
  "data": {
    "object": "order",
    "id": "ord_abc123",
    "status": "paid",
    "buyer_email": "buyer@example.com",
    "amount": 1900,
    "listing": {"id": "lst_abc123", "name": "My Ebook"},
    "unlock_url": "https://buy.listbee.so/unlock/..."
  }
}
```

## Available operations

| Resource   | Operations |
|------------|-----------|
| Bootstrap  | Start, Verify, Poll |
| Listing    | Create, Get, List, Update, Delete, Publish, Unpublish, Archive |
| Order      | Get, List, Fulfill, Refund, Redeliver |
| Account    | Get, Update, Delete |
| Stripe     | Connect, Disconnect |
| Plan       | List |

The **ListBee Trigger** node receives `order.paid`, `order.fulfilled`, `order.refunded`, and `order.disputed` events via the listing's `agent_callback_url`.
