The ListBee n8n community node lets you create listings, manage orders, and handle webhooks inside n8n workflows — no code required.

---

## Install

### From the n8n UI

1. Open your n8n instance
2. Go to **Settings → Community Nodes**
3. Search for `listbee`
4. Click **Install**

### From the command line

```bash
npm install n8n-nodes-listbee
```

Then restart n8n.

---

## Set up credentials

1. In n8n, go to **Credentials → Add Credential**
2. Search for **ListBee API**
3. Enter your API key in the **API Key** field — the value starts with `lb_`
4. Click **Save**

Get your API key at [console.listbee.so](https://console.listbee.so).

The credential sets the `Authorization: Bearer lb_...` header on all requests.

## Available operations

### Listing

| Operation | Method | Description |
|-----------|--------|-------------|
| Create | POST /v1/listings | Create a new listing |
| Get | GET /v1/listings/{slug} | Get a listing by slug |
| Update | PUT /v1/listings/{slug} | Update listing fields |
| Delete | DELETE /v1/listings/{slug} | Delete a listing |
| Get Many | GET /v1/listings | List all listings |

### Order

| Operation | Method | Description |
|-----------|--------|-------------|
| Get | GET /v1/orders/{id} | Get an order by ID |
| Get Many | GET /v1/orders | List all orders |

### Account

| Operation | Method | Description |
|-----------|--------|-------------|
| Get | GET /v1/account | Get account details and readiness |

## Node configuration: create a listing

Add a **ListBee** node to your workflow and configure it as follows:

| Field | Value |
|-------|-------|
| Resource | Listing |
| Operation | Create |
| Name | `My Product` (or expression from previous node) |
| Price | `1900` (cents — $19.00) |
| Currency | `usd` |
| Content | `https://my-bucket.s3.amazonaws.com/file.pdf` |

The node outputs the full listing object. Use `{{ $json.url }}` in downstream nodes to access the checkout URL.

**Optional fields** (click "Add Option"):

| Field | Description |
|-------|-------------|
| Description | Product description shown on checkout page |
| Slug | Custom URL slug — auto-generated from name if omitted |

## HTTP Request node alternative

If you need an operation not covered by the community node, use the **HTTP Request** node with the ListBee API directly.

**Node configuration to create a listing:**

| Field | Value |
|-------|-------|
| Method | POST |
| URL | `https://api.listbee.so/v1/listings` |
| Authentication | Header Auth |
| Header Auth Name | `Authorization` |
| Header Auth Value | `Bearer lb_...` |
| Body Content Type | JSON |
| Body | `{ "name": "My Product", "price": 1900, "currency": "usd", "content": "https://..." }` |

## Trigger: receive webhook events

Use the **Webhook** node to receive ListBee events in your workflow.

1. Add a **Webhook** node with **HTTP Method: POST**
2. Copy the webhook URL from the node
3. Register it with ListBee:

```bash
curl -X POST https://api.listbee.so/v1/webhooks \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-n8n-instance.com/webhook/listbee",
    "events": ["order.paid"]
  }'
```

When `order.paid` fires, the webhook node receives the event payload and your workflow continues.

The payload arrives at `{{ $json.body }}`:
- `{{ $json.body.type }}` — event type, e.g. `"order.paid"`
- `{{ $json.body.data.object.buyer_email }}` — buyer's email
- `{{ $json.body.data.object.id }}` — order ID

## Example workflow: notify on sale

A three-node workflow that sends a Slack message every time an order is paid.

```
Webhook (POST /webhook/listbee)
  ↓
IF  {{ $json.body.type === "order.paid" }}
  ↓
Slack — Send Message
  Channel: #sales
  Message: "New order: {{ $json.body.data.object.buyer_email }} paid
            {{ $json.body.data.object.amount / 100 }} USD"
```
