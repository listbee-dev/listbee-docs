An order is created when a buyer visits a product page. It starts as `pending` (checkout opened), transitions to `paid` on payment, and reaches `fulfilled` when delivery is complete.

---

## Order lifecycle

Orders move through five states:

```
PENDING → PAID → FULFILLED
              ↘ CANCELED
              ↘ FAILED
```

| Status | Meaning |
|--------|---------|
| `pending` | Buyer opened checkout. Payment not yet received. |
| `paid` | Payment confirmed. Managed listings are fulfilled automatically. External listings wait for your app. |
| `fulfilled` | Product delivered to the buyer. Terminal state. |
| `canceled` | Order was canceled before fulfillment. Terminal. |
| `failed` | Payment or fulfillment failed. Terminal. |

### Managed fulfillment

`PENDING → PAID → FULFILLED` happens automatically. ListBee creates an access grant and delivers the content on payment confirmation. Both `order.paid` and `order.fulfilled` webhooks fire.

### External fulfillment

`PENDING → PAID` happens automatically. Your app receives the `order.paid` webhook and handles delivery. Call `POST /v1/orders/{id}/fulfill` when done — this transitions the order to `FULFILLED` and fires `order.fulfilled`.

For physical goods, call `POST /v1/orders/{id}/ship` first to record tracking info, then `fulfill` when delivered.

---

## Fulfillment tracking

External orders have a `fulfillment_status` field that tracks delivery progress:

| `fulfillment_status` | Meaning |
|---------------------|---------|
| `pending` | Payment confirmed, waiting for fulfillment |
| `shipped` | Package shipped (after calling `/ship`) |
| `fulfilled` | Product delivered to buyer |

Managed orders skip straight to `fulfilled` — there's no intermediate state.

## Fulfill an order

`POST /v1/orders/{id}/fulfill` marks an external order as fulfilled. Optionally push generated content for ListBee to deliver.

<CodeGroup>

```python title="Python — simple fulfill"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/orders/ord_abc123/fulfill",
    headers={"Authorization": "Bearer lb_..."},
    json={},
)
order = resp.json()
print(order["status"])       # fulfilled
print(order["fulfilled_at"]) # 2026-04-02T14:30:00Z
```

```typescript title="TypeScript — simple fulfill"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const order = await client.orders.fulfill("ord_abc123", {});
console.log(order.status);       // fulfilled
console.log(order.fulfilled_at); // 2026-04-02T14:30:00Z
```

</CodeGroup>

### Fulfill with generated content

Push content to ListBee for delivery — useful for [dynamic fulfillment](/dynamic-fulfillment) (AI-generated content, custom reports).

<CodeGroup>

```python title="Python — fulfill with content"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/orders/ord_abc123/fulfill",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "content": "Your personalized horoscope: Mercury is in retrograde...",
    },
)
```

```python title="Python — fulfill with file"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/orders/ord_abc123/fulfill",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "content_url": "https://your-bucket.s3.amazonaws.com/report.pdf",
    },
)
```

```typescript title="TypeScript — fulfill with content"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

await client.orders.fulfill("ord_abc123", {
  content: "Your personalized horoscope: Mercury is in retrograde...",
});
```

</CodeGroup>

| Field | Type | Description |
|-------|------|-------------|
| `content` | string | Text content delivered inline to the buyer. |
| `content_url` | string | URL to a file or page. ListBee auto-detects file vs URL. |
| `content_type` | string | Override auto-detection: `"file"`, `"url"`, or `"text"`. |

## Ship an order

`POST /v1/orders/{id}/ship` records shipping information on a `paid` order. Use this for physical goods before marking the order as fulfilled.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/orders/ord_abc123/ship",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "carrier": "USPS",
        "tracking_code": "9400111899223456789012",
        "seller_note": "Ships from Portland. Estimated 3-5 business days.",
    },
)
order = resp.json()

print(order["fulfillment_status"])  # shipped
print(order["carrier"])             # USPS
print(order["tracking_code"])       # 9400111899223456789012
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const order = await client.orders.ship("ord_abc123", {
  carrier: "USPS",
  trackingCode: "9400111899223456789012",
  sellerNote: "Ships from Portland. Estimated 3-5 business days.",
});

console.log(order.fulfillment_status);  // shipped
console.log(order.carrier);             // USPS
console.log(order.tracking_code);       // 9400111899223456789012
```

</CodeGroup>

This fires an `order.shipped` webhook event.

## List orders

`GET /v1/orders` returns orders in reverse chronological order with cursor-based pagination.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.get(
    "https://api.listbee.so/v1/orders",
    headers={"Authorization": "Bearer lb_..."},
    params={"limit": 20},
)
result = resp.json()

for order in result["data"]:
    print(order["id"], order["status"], order["amount"])

# Paginate
if result["has_more"]:
    next_resp = httpx.get(
        "https://api.listbee.so/v1/orders",
        headers={"Authorization": "Bearer lb_..."},
        params={"limit": 20, "cursor": result["cursor"]},
    )
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const page = await client.orders.list({ limit: 20 });

for (const order of page.data) {
  console.log(order.id, order.status, order.amount);
}

if (page.has_more) {
  const nextPage = await client.orders.list({
    limit: 20,
    cursor: page.cursor,
  });
}
```

</CodeGroup>

## Get an order

`GET /v1/orders/{id}` returns a single order.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.get(
    "https://api.listbee.so/v1/orders/ord_abc123",
    headers={"Authorization": "Bearer lb_..."},
)
order = resp.json()

print(order["id"])                 # ord_abc123
print(order["status"])             # paid
print(order["amount"])             # 1900 (cents)
print(order["currency"])           # usd
print(order["listing_id"])         # lst_...
print(order["listing_slug"])       # python-async-patterns
print(order["buyer_email"])        # buyer@example.com
print(order["checkout_data"])      # { "size": "M" } or null
print(order["shipping_address"])   # { line1, city, ... } or null
print(order["fulfillment_status"]) # pending | shipped | fulfilled | null
print(order["paid_at"])            # 2026-04-02T10:00:00Z
print(order["fulfilled_at"])       # 2026-04-02T14:30:00Z or null
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const order = await client.orders.get("ord_abc123");

console.log(order.id);                 // ord_abc123
console.log(order.status);             // paid
console.log(order.amount);             // 1900 (cents)
console.log(order.currency);           // usd
console.log(order.listing_id);         // lst_...
console.log(order.listing_slug);       // python-async-patterns
console.log(order.buyer_email);        // buyer@example.com
console.log(order.checkout_data);      // { size: "M" } or null
console.log(order.shipping_address);   // { line1, city, ... } or null
console.log(order.fulfillment_status); // pending | shipped | fulfilled | null
console.log(order.paid_at);           // 2026-04-02T10:00:00Z
console.log(order.fulfilled_at);       // 2026-04-02T14:30:00Z or null
```

</CodeGroup>

## Filter orders

| Parameter | Description |
|-----------|-------------|
| `listing_slug` | Only return orders for a specific listing. |
| `status` | Filter by status: `pending`, `paid`, `fulfilled`, `canceled`, `failed`. |
| `limit` | Number of results per page. Max 100. Default 20. |
| `cursor` | Pagination cursor from previous response. |

---

## Key fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `pending`, `paid`, `fulfilled`, `canceled`, `failed` |
| `checkout_data` | object \| null | Buyer responses from checkout schema fields |
| `shipping_address` | object \| null | Structured address from `address` checkout field |
| `fulfillment_status` | string \| null | `pending`, `shipped`, `fulfilled` (external orders only) |
| `carrier` | string \| null | Shipping carrier (after `/ship` call) |
| `tracking_code` | string \| null | Tracking number (after `/ship` call) |
| `seller_note` | string \| null | Note from seller (after `/ship` call) |
| `paid_at` | string \| null | ISO 8601 timestamp of payment confirmation |
| `fulfilled_at` | string \| null | ISO 8601 timestamp of fulfillment |
