The official Python SDK wraps the ListBee REST API with a typed, synchronous and async client. All endpoints are covered, errors raise typed exceptions, and pagination uses cursor iteration.

| | |
|---|---|
| **Package** | `listbee` |
| **Install** | `pip install listbee` |
| **Min Python** | 3.9+ |
| **Auth** | `ListBee(api_key="lb_...")` or `LISTBEE_API_KEY` env var |
| **Source** | [github.com/listbee-dev/listbee-python](https://github.com/listbee-dev/listbee-python) |
| **PyPI** | [pypi.org/project/listbee](https://pypi.org/project/listbee/) |

## Install

```bash
pip install listbee
```

Requires Python 3.9+.

## Authentication

```python
from listbee import ListBee

client = ListBee(api_key="lb_...")
```

Or set the environment variable and omit the argument:

```bash
export LISTBEE_API_KEY=lb_...
```

```python
from listbee import ListBee

client = ListBee()  # reads LISTBEE_API_KEY from environment
```

## Listings

### Create

```python
listing = client.listings.create(
    name="Python Async Patterns",
    price=1900,          # cents — $19.00
    deliverable="https://my-bucket.s3.amazonaws.com/python-async.pdf",
)

print(listing.url)               # https://buy.listbee.so/python-async-patterns
print(listing.readiness.sellable)  # True
```

Optional fields:

```python
listing = client.listings.create(
    name="SQL Performance Guide",
    price=2900,
    deliverable="https://my-bucket.s3.amazonaws.com/sql-guide.pdf",
    description="50 query patterns for high-traffic Postgres databases.",
    slug="sql-performance-guide",  # auto-generated if omitted
)
```

### List

```python
page = client.listings.list(limit=20)

for listing in page.data:
    print(listing.slug, listing.price, listing.readiness.sellable)

# Next page
if page.has_more:
    next_page = client.listings.list(cursor=page.cursor, limit=20)
```

### Get

```python
listing = client.listings.get("python-async-patterns")  # slug, not ID

print(listing.id)    # lst_...
print(listing.url)   # https://buy.listbee.so/python-async-patterns
```

### Update

```python
listing = client.listings.update(
    "python-async-patterns",
    price=2400,
    description="Updated description.",
)

print(listing.price)  # 2400
```

Only the fields you pass are updated.

### Delete

```python
client.listings.delete("python-async-patterns")
# Returns None on success
```

## Error handling

All errors raise a subclass of `ListBeeError`. Catch specific exceptions for fine-grained handling.

```python
from listbee import ListBee
from listbee.exceptions import (
    ListBeeError,
    AuthenticationError,
    NotFoundError,
    RateLimitError,
    ValidationError,
)

client = ListBee(api_key="lb_...")

try:
    listing = client.listings.get("nonexistent-slug")
except NotFoundError as e:
    print(f"Not found: {e.detail}")
except AuthenticationError as e:
    print(f"Invalid API key: {e.detail}")
except RateLimitError as e:
    print(f"Rate limited. Retry after: {e.retry_after}s")
except ValidationError as e:
    print(f"Validation error on field '{e.param}': {e.detail}")
except ListBeeError as e:
    print(f"API error {e.status}: {e.detail}")
```

All exceptions expose `.status` (HTTP status code), `.detail` (human-readable message), and `.code` (machine-readable error code from the API).

## Pagination

List methods return a page object with `.data`, `.has_more`, and `.cursor`. Iterate manually or use the helper:

```python
# Manual cursor iteration
cursor = None
while True:
    page = client.listings.list(cursor=cursor, limit=100)
    for listing in page.data:
        process(listing)
    if not page.has_more:
        break
    cursor = page.cursor

# Auto-iterate all pages
for listing in client.listings.iter_all():
    process(listing)
```

`iter_all()` is available on listings and orders resources.

## Async client

Use `AsyncListBee` for async contexts:

```python
import asyncio
from listbee import AsyncListBee

async def main():
    client = AsyncListBee(api_key="lb_...")

    listing = await client.listings.create(
        name="Python Async Patterns",
        price=1900,
        deliverable="https://my-bucket.s3.amazonaws.com/python-async.pdf",
    )
    print(listing.url)

    # List with pagination
    page = await client.listings.list(limit=20)
    for l in page.data:
        print(l.slug)

asyncio.run(main())
```

All methods on `AsyncListBee` are `async` and return the same typed objects as the sync client.

## Common errors

| Error | Cause | Resolution |
|-------|-------|------------|
| `AuthenticationError` | API key missing, invalid, or revoked | Pass a valid `lb_...` key to `ListBee()` or set `LISTBEE_API_KEY` env var |
| `NotFoundError` | Listing slug or order ID doesn't exist | Verify the slug/ID — listings use slug (`r7kq2xy9`), orders use ID (`ord_...`) |
| `ValidationError` | Request body failed server-side validation | Check the `param` attribute on the error for the offending field |
| `RateLimitError` | Too many requests | The SDK does not auto-retry on 429. Implement backoff using `retry_after` attribute |
| `ImportError: No module named 'listbee'` | Package not installed | Run `pip install listbee` in the correct virtual environment |

## Limitations

- **No auto-retry** — the SDK raises on 429/5xx. Implement your own retry logic.
- **No webhook signature verification** — use `hmac` directly as shown in the [Webhooks](/webhooks) guide.
- **No file upload** — the `content` field accepts URLs, not file objects. Upload to your own storage first.
- **Sync by default** — use `AsyncListBee` for async contexts. The sync and async clients have identical APIs.
