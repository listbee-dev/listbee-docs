Three steps to go from zero to a live listing: create an account, connect Stripe, create a listing.

---

## Step 1: Create an account

Send your email to receive a one-time code. Then verify the code to get your API key.

```bash
# 1a. Request OTP
curl -X POST https://api.listbee.so/v1/auth/otp/start \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com"}'
# → 200 { "object": "signup_session", "email": "you@example.com", "status": "otp_sent" }
```

Check your inbox for the 6-digit code, then:

```bash
# 1b. Verify and get API key
curl -X POST https://api.listbee.so/v1/auth/otp/verify \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "code": "123456"}'
# → 201 { "object": "signup_result", "account": { "id": "acc_..." }, "api_key": "lb_..." }
```

Store the API key — all subsequent requests use it as a Bearer token.

---

## Step 2: Connect Stripe

Two options depending on whether you already have a Stripe account.

### Option A: Submit a Stripe key (API-driven, no browser)

```bash
curl -X POST https://api.listbee.so/v1/account/stripe-key \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"secret_key": "sk_live_..."}'
# → 200 account object with readiness.operational: true
```

### Option B: Stripe Connect onboarding (browser required)

```bash
curl -X POST https://api.listbee.so/v1/account/stripe/connect \
  -H "Authorization: Bearer lb_..."
# → 200 { "object": "stripe_connect_session", "url": "https://connect.stripe.com/..." }
```

Open the URL in a browser to complete Stripe setup. After completion, `GET /v1/account` will show `readiness.operational: true`.

---

## Step 3: Create a listing

Create a draft, attach a deliverable, then publish.

```bash
# 3a. Create a draft listing
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Python Async Patterns",
    "price": 1900
  }'
# → 201 { "object": "listing", "id": "lst_...", "slug": "python-async-patterns", "status": "draft" }
```

```bash
# 3b. Attach a deliverable (file upload or URL/text)
curl -X PUT https://api.listbee.so/v1/listings/lst_.../deliverable \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "type": "url",
    "value": "https://your-bucket.s3.amazonaws.com/python-async-patterns.pdf"
  }'
# → 200 listing with deliverable attached, fulfillment: "managed"
```

```bash
# 3c. Publish — listing goes live
curl -X POST https://api.listbee.so/v1/listings/lst_.../publish \
  -H "Authorization: Bearer lb_..."
# → 200 { "status": "published", "url": "https://buy.listbee.so/python-async-patterns", "readiness": { "sellable": true } }
```

Share the `url` with buyers. When `readiness.sellable` is `true`, the listing is live and accepting payments.

### External fulfillment (your app delivers)

Skip the deliverable step. Register a webhook and publish without content — ListBee collects payment and fires `order.paid`.

```bash
# Register a webhook
curl -X POST https://api.listbee.so/v1/webhooks \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Order notifications",
    "url": "https://yourapp.com/webhooks/listbee",
    "events": ["order.paid"]
  }'

# Create and publish (no deliverable = external fulfillment)
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "Custom Portrait Drawing", "price": 4900}'

curl -X POST https://api.listbee.so/v1/listings/lst_.../publish \
  -H "Authorization: Bearer lb_..."
```

```bash
#   POST /v1/account/stripe-key       { "secret_key": "sk_live_..." }  → operational

#
# Step 3: Listing (draft → deliverable → publish)
#   POST /v1/listings                          { name, price }           → draft
#   PUT  /v1/listings/{id}/deliverable         { type, value/token }     → managed
#   POST /v1/listings/{id}/publish             → published, sellable
#
# External (no deliverable):
#   POST /v1/webhooks  { name, url, events: ["order.paid"] }
#   POST /v1/listings  { name, price }  → POST .../publish
#
# Auth: Authorization: Bearer lb_...
# Errors: RFC 9457 { type, title, status, detail, code, param }
# Docs: https://docs.listbee.so
```
