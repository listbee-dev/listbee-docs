## Step 1: Create an account

No console signup needed. The agent bootstraps directly via the API. See [Authentication](/authentication) for the full flow.

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import ListBee

bootstrapping = ListBee()
api_key = bootstrapping.bootstrap.run(
    "seller@example.com",
    on_otp=lambda: input("Enter OTP from email: "),
    on_human_action=lambda url: print(f"Complete Stripe: {url}"),
)

client = ListBee(api_key=api_key)
```
</CodeBlock>
<CodeBlock title="curl">
```bash
# 1. Send OTP
curl -X POST https://api.listbee.so/v1/bootstrap/start \
  -H "Content-Type: application/json" \
  -d '{"email": "seller@example.com"}'
# → {"bootstrap_token": "bst_...", "account_id": "acc_...", ...}

# 2. Human pastes the 6-digit code from email
curl -X POST https://api.listbee.so/v1/bootstrap/verify \
  -H "Content-Type: application/json" \
  -d '{"bootstrap_token": "bst_...", "otp_code": "123456"}'
# → {"api_key": "lb_...", "stripe_onboarding_url": "...", ...}

# 3. Human opens stripe_onboarding_url, agent polls until ready
curl https://api.listbee.so/v1/bootstrap/acc_... \
  -H "Authorization: Bearer lb_..."
# → {"ready": true, ...}
```
</CodeBlock>
</CodeBlocks>

Save the `api_key` — it's shown once. Use it as `Authorization: Bearer lb_...` on every subsequent request.

## Step 2: Create and publish a listing

A listing is a sellable item with a hosted product page. Required: `name` and `price` (in cents).

Pick a fulfillment mode:

- **`static`** — bake the deliverable into the listing. ListBee delivers it automatically at payment.
- **`async`** — leave the deliverable empty. Your agent generates and pushes it after each order via `/fulfill`.

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import Deliverable

created = client.listings.create(
    name="SEO Playbook 2026",
    price=1900,                             # $19.00
    fulfillment_mode="static",
    deliverable=Deliverable.url("https://example.com/seo-playbook.pdf"),
)

listing = created.listing
signing_secret = created.signing_secret     # store this — webhook verification

client.listings.publish(listing_id=listing.id)
print(listing.url)                          # share with buyers
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "SEO Playbook 2026",
    "price": 1900,
    "fulfillment_mode": "static",
    "deliverable": {"type": "url", "value": "https://example.com/seo-playbook.pdf"}
  }'

# Then publish:
curl -X POST https://api.listbee.so/v1/listings/lst_.../publish \
  -H "Authorization: Bearer lb_..."
```
</CodeBlock>
</CodeBlocks>

`signing_secret` is the HMAC key for webhook payloads on this listing. It's returned once at creation — store it now. See [Fulfillment](/fulfillment) for webhook signing.

## Step 3: Handle orders

When a buyer pays, ListBee creates an order (`status: paid`) and fires `order.paid`. For `static` listings, ListBee also delivers the pre-baked `deliverable`, transitions the order to `fulfilled`, and fires `order.fulfilled`.

For `async` listings, your agent generates content after payment and pushes it back:

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import Deliverable

# In your order.paid handler:
client.orders.fulfill(
    order_id="ord_9xM4kP7nR2qT5wY1",
    deliverable=Deliverable.text("Your personalized report: ..."),
)
```
</CodeBlock>
<CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/orders/ord_9xM4kP7nR2qT5wY1/fulfill \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"deliverable": {"type": "text", "value": "Your personalized report: ..."}}'
```
</CodeBlock>
</CodeBlocks>

ListBee emails the buyer a permanent `unlock_url` and transitions the order to `fulfilled`.

## Step 4: Receive webhooks

Pass `agent_callback_url` on the listing to have ListBee POST events to your server. Events are signed with the listing's `signing_secret` (HMAC-SHA256).

```python
client.listings.update(
    listing_id=listing.id,
    agent_callback_url="https://your-agent.example.com/hooks/listbee",
)
```

No URL? You can still poll `GET /v1/orders` or `GET /v1/events` for a reconciliation pass. See [Events](/events) and [Fulfillment](/fulfillment).
