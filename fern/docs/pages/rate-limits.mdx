Each endpoint has its own rate limit. Limits are applied per API key.

---

## Limits by endpoint

| Endpoint | Limit |
|----------|-------|
| `POST /v1/listings` | 60 / min |
| `PUT /v1/listings/{slug}` | 60 / min |
| `POST /v1/api-keys` | 10 / min |
| `POST /v1/account/signup` | 10 / hour |
| `POST /v1/account/verify` | 10 / hour |
| `POST /v1/account/stripe-key` | 10 / min |
| `POST /v1/account/stripe/connect` | 5 / min |
| `GET` endpoints | 120 / min |

---

## Rate limit response

When you exceed a limit, the API returns `429 Too Many Requests` with a `Retry-After` header:

```http
HTTP/1.1 429 Too Many Requests
Retry-After: 14
Content-Type: application/problem+json

{
  "type": "https://docs.listbee.so/errors/rate-limit-exceeded",
  "title": "Too many requests",
  "status": 429,
  "detail": "Rate limit exceeded. Retry after 14 seconds.",
  "code": "rate_limit_exceeded",
  "param": null
}
```

The `Retry-After` value is in seconds. Wait that long before retrying.

---

## Handling rate limits

Use exponential backoff with jitter. Do not retry immediately — hammering after a 429 prolongs the backoff window.

```python
import httpx
import time
import random

BASE = "https://api.listbee.so"
API_KEY = "lb_..."


def create_listing_with_backoff(payload: dict, max_attempts: int = 5) -> dict:
    delay = 1.0

    for attempt in range(max_attempts):
        resp = httpx.post(
            f"{BASE}/v1/listings",
            headers={"Authorization": f"Bearer {API_KEY}"},
            json=payload,
        )

        if resp.status_code == 429:
            retry_after = int(resp.headers.get("Retry-After", delay))
            # Use whichever is larger: server hint or exponential backoff
            wait = max(retry_after, delay) + random.uniform(0, 1)
            print(f"Rate limited. Waiting {wait:.1f}s (attempt {attempt + 1})")
            time.sleep(wait)
            delay *= 2  # exponential backoff
            continue

        resp.raise_for_status()
        return resp.json()

    raise RuntimeError("Max retry attempts reached")


listing = create_listing_with_backoff({
    "name": "Python Async Patterns",
    "price": 1900,
    "currency": "usd",
    "content": "https://my-bucket.s3.amazonaws.com/python-async.pdf",
})
print(listing["url"])
```

---

## Tips for staying within limits

- **Batch reads before writes.** Use `GET /v1/listings` to fetch existing listings before creating new ones.
- **Cache responses.** Listing and account data rarely change — cache `GET` responses for 30–60 seconds.
- **Avoid polling tight loops.** When waiting for `readiness.operational`, use `time.sleep(5)` between polls, not `time.sleep(0.1)`.
- **Use idempotency keys.** Retrying with the same `Idempotency-Key` hits the cache, not the rate limiter.
