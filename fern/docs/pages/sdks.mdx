ListBee ships first-party SDKs for Python and TypeScript. Both are thin wrappers over the REST API with typed responses, automatic retries, and cursor-based pagination helpers.

## Python

### Install

```bash
pip install listbee
```

Requires Python 3.9+.

### Auth

```python
from listbee import ListBee

client = ListBee(api_key="lb_...")
```

Pass `api_key` explicitly, or set the `LISTBEE_API_KEY` environment variable and call `ListBee()` with no arguments.

### Create a listing

```python
from listbee import ListBee

client = ListBee(api_key="lb_...")

listing = client.listings.create(
    name="Python Fundamentals",
    price=2900,           # $29.00
)

print(listing.id)     # lst_abc123...
print(listing.url)    # https://buy.listbee.so/aB3xK9m
```

### List orders

```python
orders = client.orders.list(limit=20)

for order in orders.data:
    print(order.id, order.status, order.buyer_email)

# Fetch next page
if orders.has_more:
    next_page = client.orders.list(limit=20, cursor=orders.cursor)
```

## TypeScript

### Install

```bash
npm install listbee
```

Requires Node.js 18+. Works in Deno and Bun.

### Auth

```typescript
import { ListBee } from 'listbee';

const client = new ListBee({ apiKey: 'lb_...' });
```

Or set `LISTBEE_API_KEY` in your environment and omit the argument.

### Create a listing

```typescript
import { ListBee } from 'listbee';

const client = new ListBee({ apiKey: 'lb_...' });

const listing = await client.listings.create({
  name: 'TypeScript Handbook',
  price: 1900,           // $19.00
});

console.log(listing.id);    // lst_abc123...
console.log(listing.url);   // https://buy.listbee.so/tK9mN3p
```

### List orders

```typescript
const orders = await client.orders.list({ limit: 20 });

for (const order of orders.data) {
  console.log(order.id, order.status, order.buyer_email);
}

// Fetch next page
if (orders.has_more) {
  const nextPage = await client.orders.list({ limit: 20, cursor: orders.cursor });
}
```

## Error handling

Both SDKs raise typed exceptions for API errors. Every exception includes an `extras` dict/object containing RFC 9457 extension members — structured metadata that helps your code react to specific failure modes.

<CodeBlocks>
<CodeBlock title="Python">
```python
from listbee import ListBee
from listbee.exceptions import ConflictError, RateLimitError

client = ListBee(api_key="lb_...")

try:
    client.listings.publish(listing_id="lst_abc123")
except ConflictError as e:
    if e.code == "publish_failed":
        # Readiness actions tell you exactly what to fix
        for action in e.extras.get("actions", []):
            print(f"→ {action['message']}")
    elif e.code == "not_in_draft":
        print(f"Status is {e.extras['current_status']}, need draft")
except RateLimitError as e:
    wait = e.extras.get("retry_after", 10)
    print(f"Rate limited — retry in {wait}s")
```
</CodeBlock>
<CodeBlock title="TypeScript">
```typescript
import { ListBee, ConflictError, RateLimitError } from 'listbee';

const client = new ListBee({ apiKey: 'lb_...' });

try {
  await client.listings.publish('lst_abc123');
} catch (e) {
  if (e instanceof ConflictError && e.code === 'publish_failed') {
    const actions = e.extras.actions as Array<{ message: string }>;
    actions?.forEach(a => console.log(`→ ${a.message}`));
  } else if (e instanceof RateLimitError) {
    const wait = (e.extras.retry_after as number) ?? 10;
    console.log(`Rate limited — retry in ${wait}s`);
  }
}
```
</CodeBlock>
</CodeBlocks>

See [Errors](/errors#extension-members) for the full list of enriched error codes and their extension members.

## curl reference

All SDK methods map 1:1 to REST endpoints. Any example can be run with curl:

```bash
# Create a listing
curl -X POST https://api.listbee.so/v1/listings \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "Python Fundamentals", "price": 2900}'

# List orders
curl https://api.listbee.so/v1/orders \
  -H "Authorization: Bearer lb_..."
```
