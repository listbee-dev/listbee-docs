title: Stores

Stores are brand containers. Each store has its own handle, payment config, and optional custom domain. A default store is auto-created on signup — single-store accounts don't need to think about stores at all.

---

## Creating a store

`POST /v1/stores` requires `handle` (unique, URL-safe identifier). `name` is optional — if omitted, handle is used as the display name.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/stores",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "handle": "fitness-brand",
        "name": "Fitness Brand",
    },
)
store = resp.json()

print(store["id"])      # str_...
print(store["handle"])  # fitness-brand
print(store["object"])  # store
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/stores", {
  method: "POST",
  headers: {
    "Authorization": "Bearer lb_...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    handle: "fitness-brand",
    name: "Fitness Brand",
  }),
});
const store = await resp.json();

console.log(store.id);      // str_...
console.log(store.handle);  // fitness-brand
console.log(store.object);  // store
```

</CodeGroup>

The response is a store object. `id` uses the `str_` prefix. `payment_connected` will be `false` until you connect Stripe.

## Listing stores

`GET /v1/stores` returns all stores for the account. This endpoint is non-paginated — accounts have a bounded number of stores.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.get(
    "https://api.listbee.so/v1/stores",
    headers={"Authorization": "Bearer lb_..."},
)
result = resp.json()

for store in result["data"]:
    print(store["id"], store["handle"], store["payment_connected"])
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/stores", {
  headers: { "Authorization": "Bearer lb_..." },
});
const result = await resp.json();

for (const store of result.data) {
  console.log(store.id, store.handle, store.payment_connected);
}
```

</CodeGroup>

## Getting a store

`GET /v1/stores/{id}` returns a single store by ID.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.get(
    "https://api.listbee.so/v1/stores/str_...",
    headers={"Authorization": "Bearer lb_..."},
)
store = resp.json()
print(store["display_name"])
print(store["payment_connected"])
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/stores/str_...", {
  headers: { "Authorization": "Bearer lb_..." },
});
const store = await resp.json();
console.log(store.display_name);
console.log(store.payment_connected);
```

</CodeGroup>

## Updating a store

`PATCH /v1/stores/{id}` accepts a partial update. Only the fields you include are changed. Updatable fields: `name`, `bio`, `social_links`.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.patch(
    "https://api.listbee.so/v1/stores/str_...",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "Fitness Brand Pro",
        "bio": "Premium fitness programs and guides.",
        "social_links": {
            "twitter": "https://twitter.com/fitnessbrand",
        },
    },
)
store = resp.json()
print(store["display_name"])  # Fitness Brand Pro
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/stores/str_...", {
  method: "PATCH",
  headers: {
    "Authorization": "Bearer lb_...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Fitness Brand Pro",
    bio: "Premium fitness programs and guides.",
    social_links: {
      twitter: "https://twitter.com/fitnessbrand",
    },
  }),
});
const store = await resp.json();
console.log(store.display_name);  // Fitness Brand Pro
```

</CodeGroup>

## Deleting a store

`DELETE /v1/stores/{id}` removes the store. The default store cannot be deleted.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.delete(
    "https://api.listbee.so/v1/stores/str_...",
    headers={"Authorization": "Bearer lb_..."},
)
# 204 No Content on success
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/stores/str_...", {
  method: "DELETE",
  headers: { "Authorization": "Bearer lb_..." },
});
// 204 No Content on success
```

</CodeGroup>

## Connecting Stripe to a store

`POST /v1/stores/{id}/stripe/connect` starts a Stripe Connect onboarding flow for the store. Returns a URL the user opens in their browser to complete setup.

Payment config is per-store. Each store connects its own Stripe account independently.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/stores/str_.../stripe/connect",
    headers={"Authorization": "Bearer lb_..."},
)
result = resp.json()
print(result["url"])  # https://connect.stripe.com/... — open in browser
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/stores/str_.../stripe/connect", {
  method: "POST",
  headers: { "Authorization": "Bearer lb_..." },
});
const result = await resp.json();
console.log(result.url);  // https://connect.stripe.com/... — open in browser
```

</CodeGroup>

After the user completes onboarding, `payment_connected` will be `true` on the store object.

## Multi-store setup

The default store is created automatically on signup. Single-store accounts work without ever specifying a store — it's always implied.

For multi-store accounts, pass `store_id` when creating listings to assign them to a specific store:

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/listings",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "name": "30-Day Strength Program",
        "price": 2900,
        "content": "https://your-bucket.s3.amazonaws.com/strength-program.pdf",
        "store_id": "str_...",  # assign to specific store
    },
)
listing = resp.json()
print(listing["store_id"])  # str_...
```

```typescript title="TypeScript"
const resp = await fetch("https://api.listbee.so/v1/listings", {
  method: "POST",
  headers: {
    "Authorization": "Bearer lb_...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "30-Day Strength Program",
    price: 2900,
    content: "https://your-bucket.s3.amazonaws.com/strength-program.pdf",
    store_id: "str_...",  // assign to specific store
  }),
});
const listing = await resp.json();
console.log(listing.store_id);  // str_...
```

</CodeGroup>

Single-store accounts can omit `store_id` — the listing is automatically assigned to the default store.

Each store has independent payment config. Buyers purchasing a listing from store A are charged through store A's Stripe account, not store B's.
