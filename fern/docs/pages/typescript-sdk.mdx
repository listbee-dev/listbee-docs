The official TypeScript SDK wraps the ListBee REST API with a fully typed client. All endpoints are covered, errors raise typed exceptions, and pagination uses cursor-based iteration.

| | |
|---|---|
| **Package** | `listbee` |
| **Install** | `npm install listbee` |
| **Min Node** | 18+ |
| **Auth** | `new ListBee({ apiKey: 'lb_...' })` or `LISTBEE_API_KEY` env var |
| **Source** | [github.com/listbee-dev/listbee-typescript](https://github.com/listbee-dev/listbee-typescript) |
| **npm** | [npmjs.com/package/listbee](https://www.npmjs.com/package/listbee) |

---

## Install

```bash
npm install listbee
```

Requires Node.js 18+.

## Authentication

```typescript
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });
```

Or set the environment variable and omit the argument:

```bash
export LISTBEE_API_KEY=lb_...
```

```typescript
import { ListBee } from "listbee";

const client = new ListBee(); // reads LISTBEE_API_KEY from environment
```

## Listings

### Create

```typescript
const listing = await client.listings.create({
  name: "Python Async Patterns",
  price: 1900,          // cents — $19.00
  content: "https://my-bucket.s3.amazonaws.com/python-async.pdf",
});

console.log(listing.url);               // https://buy.listbee.so/python-async-patterns
console.log(listing.readiness.sellable); // true
```

Optional fields:

```typescript
const listing = await client.listings.create({
  name: "SQL Performance Guide",
  price: 2900,
  content: "https://my-bucket.s3.amazonaws.com/sql-guide.pdf",
  description: "50 query patterns for high-traffic Postgres databases.",
  slug: "sql-performance-guide",  // auto-generated if omitted
});
```

### List

```typescript
const page = await client.listings.list({ limit: 20 });

for (const listing of page.data) {
  console.log(listing.slug, listing.price, listing.readiness.sellable);
}

// Next page
if (page.has_more) {
  const nextPage = await client.listings.list({ cursor: page.cursor, limit: 20 });
}
```

### Get

```typescript
const listing = await client.listings.get("python-async-patterns"); // slug, not ID

console.log(listing.id);   // lst_...
console.log(listing.url);  // https://buy.listbee.so/python-async-patterns
```

### Update

```typescript
const listing = await client.listings.update("python-async-patterns", {
  price: 2400,
  description: "Updated description.",
});

console.log(listing.price); // 2400
```

Only the fields you pass are updated.

### Pause and resume

```typescript
const paused = await client.listings.pause("python-async-patterns");
console.log(paused.status); // "paused"

const active = await client.listings.resume("python-async-patterns");
console.log(active.status); // "active"
```

### Delete

```typescript
await client.listings.delete("python-async-patterns");
// Returns void on success
```

## Orders

### List

```typescript
const page = await client.orders.list({ limit: 20 });

for (const order of page.data) {
  console.log(order.id, order.buyer_email, order.status);
}
```

Filter by status, listing, or date range:

```typescript
const page = await client.orders.list({
  status: "paid",
  listing: "python-async-patterns",
  created_after: new Date("2026-01-01"),
  created_before: new Date("2026-03-31"),
  limit: 50,
});
```

### Get

```typescript
const order = await client.orders.get("ord_...");

console.log(order.id);            // ord_...

console.log(order.buyer_email);
console.log(order.status);        // "paid" | "pending" | "expired"
console.log(order.amount);        // cents
```

## Signup

Signup endpoints are unauthenticated — no API key needed.

```typescript
import { ListBee } from "listbee";

const client = new ListBee(); // no apiKey required for signup

// Request OTP
await client.signup.create({ email: "you@example.com" });

// Verify and receive API key
const result = await client.signup.verify({
  email: "you@example.com",
  code: "123456",
});

console.log(result.api_key);       // lb_... — store securely
console.log(result.account.id);    // acc_...
```

## Error handling

All errors throw a subclass of `ListBeeError`. Catch specific exceptions for fine-grained handling.

```typescript
import { ListBee } from "listbee";
import {
  ListBeeError,
  AuthenticationError,
  NotFoundError,
  RateLimitError,
  ValidationError,
} from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

try {
  const listing = await client.listings.get("nonexistent-slug");
} catch (e) {
  if (e instanceof NotFoundError) {
    console.error(`Not found: ${e.detail}`);
  } else if (e instanceof AuthenticationError) {
    console.error(`Invalid API key: ${e.detail}`);
  } else if (e instanceof RateLimitError) {
    console.error(`Rate limited. Limit: ${e.limit}, reset: ${e.reset}`);
  } else if (e instanceof ValidationError) {
    console.error(`Validation error on field '${e.param}': ${e.detail}`);
  } else if (e instanceof ListBeeError) {
    console.error(`API error ${e.status}: ${e.detail}`);
  }
}
```

All exceptions expose `.status` (HTTP status code), `.detail` (human-readable message), `.code` (machine-readable error code), `.type`, `.title`, and `.param`. `RateLimitError` adds `.limit`, `.remaining`, and `.reset`.

## Pagination

List methods return a `CursorPage` object with `.data`, `.has_more`, and `.cursor`. Iterate manually:

```typescript
// Manual cursor iteration
let cursor: string | undefined;
while (true) {
  const page = await client.listings.list({ cursor, limit: 100 });
  for (const listing of page.data) {
    process(listing);
  }
  if (!page.has_more) break;
  cursor = page.cursor;
}
```

## Common errors

| Error | Cause | Resolution |
|-------|-------|------------|
| `AuthenticationError` | API key missing, invalid, or revoked | Pass a valid `lb_...` key to `new ListBee()` or set `LISTBEE_API_KEY` env var |
| `NotFoundError` | Listing slug or order ID doesn't exist | Verify the slug/ID — listings use slug (`r7kq2xy9`), orders use ID (`ord_...`) |
| `ValidationError` | Request body failed server-side validation | Check the `param` attribute on the error for the offending field |
| `RateLimitError` | Too many requests | The SDK does not auto-retry on 429. Implement backoff using the `reset` attribute |
| `Cannot find module 'listbee'` | Package not installed | Run `npm install listbee` in your project |

## Limitations

- **No auto-retry** — the SDK throws on 429/5xx. Implement your own retry logic.
- **No webhook signature verification** — use `crypto.createHmac` directly as shown in the [Webhooks](/webhooks) guide.
- **No file upload** — the `content` field accepts URLs, not file objects. Upload to your own storage first.
