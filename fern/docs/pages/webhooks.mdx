Webhooks let you receive event notifications over HTTP. When an event occurs — a buyer pays, an order is fulfilled — ListBee sends a signed POST request to your endpoint.

---

## Register a webhook

`POST /v1/webhooks` takes a URL and a list of event types to subscribe to.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/webhooks",
    headers={"Authorization": "Bearer lb_..."},
    json={
        "url": "https://yourapp.com/webhooks/listbee",
        "events": ["order.paid", "order.fulfilled", "listing.created"],
    },
)
webhook = resp.json()

print(webhook["id"])      # wh_...
print(webhook["secret"])  # whsec_... — store this, it's shown once
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const webhook = await client.webhooks.create({
  url: "https://yourapp.com/webhooks/listbee",
  events: ["order.paid", "order.fulfilled", "listing.created"],
});

console.log(webhook.id);      // wh_...
console.log(webhook.secret);  // whsec_... — store this, it's shown once
```

</CodeGroup>

The `secret` field is returned only at creation time. Store it — you need it to verify incoming requests.

## Events

| Event | When it fires |
|-------|--------------|
| `order.created` | A buyer opens checkout (order is `pending`). |
| `order.paid` | Payment confirmed. Fires for both managed and external fulfillment. |
| `order.fulfilled` | Order marked as fulfilled. Managed: fires immediately after `order.paid`. External: fires after you call `/fulfill`. |
| `order.shipped` | Shipping info recorded via `POST /v1/orders/{id}/ship`. |
| `listing.created` | A new listing is created. |
| `listing.updated` | Any field on a listing changes. |
| `listing.deleted` | A listing is deleted. |

Subscribe to the events you care about. You can subscribe to multiple events per webhook endpoint.

<Note>
`order.paid` and `order.fulfilled` both fire for all fulfillment modes. For managed listings, they fire in quick succession. For external listings, `order.fulfilled` fires only after you call the fulfill endpoint.
</Note>

## Payload structure

Every webhook request has the same envelope:

```json
{
  "id": "evt_...",
  "object": "event",
  "type": "order.paid",
  "created_at": "2026-04-02T10:30:00Z",
  "data": {
    "object": { ... }
  }
}
```

`data.object` contains the full resource that triggered the event — a listing object for listing events, an order object for order events.

## Signature verification

Every webhook request includes a `X-ListBee-Signature` header. Verify it before processing the payload.

The signature is an HMAC-SHA256 hex digest of the raw request body, signed with your webhook secret.

<CodeGroup>

```python title="Python"
import hashlib
import hmac
from fastapi import Request, HTTPException

WEBHOOK_SECRET = "whsec_..."  # from webhook creation

async def handle_webhook(request: Request):
    payload = await request.body()
    signature = request.headers.get("X-ListBee-Signature")

    expected = hmac.new(
        WEBHOOK_SECRET.encode(),
        payload,
        hashlib.sha256,
    ).hexdigest()

    if not hmac.compare_digest(expected, signature):
        raise HTTPException(status_code=400, detail="Invalid signature")

    event = await request.json()
    event_type = event["type"]

    if event_type == "order.paid":
        order = event["data"]["object"]
        print(f"Order paid: {order['id']} — {order['buyer_email']}")

    elif event_type == "order.fulfilled":
        order = event["data"]["object"]
        print(f"Order fulfilled: {order['id']}")

    elif event_type == "order.shipped":
        order = event["data"]["object"]
        print(f"Order shipped: {order['id']} via {order['carrier']}")

    return {"ok": True}
```

```typescript title="TypeScript"
import { createHmac, timingSafeEqual } from "crypto";
import type { Request, Response } from "express";

const WEBHOOK_SECRET = "whsec_..."; // from webhook creation

export function handleWebhook(req: Request, res: Response) {
  const payload = req.body as Buffer; // ensure raw body
  const signature = req.headers["x-listbee-signature"] as string;

  const expected = createHmac("sha256", WEBHOOK_SECRET)
    .update(payload)
    .digest("hex");

  const sigBuffer = Buffer.from(signature, "hex");
  const expectedBuffer = Buffer.from(expected, "hex");

  if (
    sigBuffer.length !== expectedBuffer.length ||
    !timingSafeEqual(sigBuffer, expectedBuffer)
  ) {
    return res.status(400).json({ error: "Invalid signature" });
  }

  const event = JSON.parse(payload.toString());

  if (event.type === "order.paid") {
    const order = event.data.object;
    console.log(`Order paid: ${order.id} — ${order.buyer_email}`);
  } else if (event.type === "order.fulfilled") {
    const order = event.data.object;
    console.log(`Order fulfilled: ${order.id}`);
  } else if (event.type === "order.shipped") {
    const order = event.data.object;
    console.log(`Order shipped: ${order.id} via ${order.carrier}`);
  }

  res.json({ ok: true });
}
```

</CodeGroup>

<Warning>
Always use timing-safe comparison for signature verification. Standard string equality (`==`) is vulnerable to timing attacks that can leak your webhook secret.
</Warning>

## Retry policy

ListBee retries failed deliveries with exponential backoff. A delivery is considered failed if your endpoint returns a non-2xx status code or does not respond within 10 seconds.

| Attempt | Delay |
|---------|-------|
| 1 | Immediately |
| 2 | 30 seconds |
| 3 | 5 minutes |
| 4 | 30 minutes |
| 5 | 2 hours |

After 5 failed attempts, the event is dropped. Your endpoint should return `200` as quickly as possible — process the event asynchronously if needed.

## Test an endpoint

Send a test event to verify your endpoint is reachable and signature verification works.

<CodeGroup>

```python title="Python"
import httpx

resp = httpx.post(
    "https://api.listbee.so/v1/webhooks/wh_.../test",
    headers={"Authorization": "Bearer lb_..."},
)
result = resp.json()
print(result["delivered"])    # True if your endpoint returned 2xx
print(result["status_code"])  # HTTP status your endpoint returned
```

```typescript title="TypeScript"
import { ListBee } from "listbee";

const client = new ListBee({ apiKey: "lb_..." });

const result = await client.webhooks.test("wh_...");
console.log(result.delivered);    // true if your endpoint returned 2xx
console.log(result.status_code);  // HTTP status your endpoint returned
```

</CodeGroup>
