Webhooks deliver event notifications to your server when things happen in ListBee — a buyer pays, an order is fulfilled, a listing goes out of stock. Your endpoint receives an HTTP POST with the event payload. No polling required.

## How it works

Register an HTTPS endpoint via `POST /v1/webhooks`. ListBee sends a POST request to that URL when subscribed events occur. Each request includes an HMAC-SHA256 signature header for verification.

Delivery is attempted with exponential backoff. If all retries are exhausted, the event is marked `failed` and the webhook may be auto-disabled after consecutive failures. Disabled webhooks stop receiving events — check `readiness.actions` on your listings for a `webhook_disabled` action.

All order event payloads include the full order snapshot. No follow-up API call is needed to get order details.

## Registering an endpoint

<CodeBlocks>
  <CodeBlock title="Python">
```python
from listbee import ListBee

client = ListBee(api_key="lb_...")

webhook = client.webhooks.create(
    name="Order handler",
    url="https://your-server.com/webhooks/listbee",
    events=["order.paid", "order.fulfilled"],
)

print(webhook.id)      # wh_7kQ2xY9mN3pR5tW1vB8a
print(webhook.secret)  # whsec_... (store this — shown once)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST https://api.listbee.so/v1/webhooks \
  -H "Authorization: Bearer lb_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Order handler",
    "url": "https://your-server.com/webhooks/listbee",
    "events": ["order.paid", "order.fulfilled"]
  }'
```
  </CodeBlock>
</CodeBlocks>

Response:

```json
{
  "object": "webhook",
  "id": "wh_7kQ2xY9mN3pR5tW1vB8a",
  "name": "Order handler",
  "url": "https://your-server.com/webhooks/listbee",
  "secret": "whsec_abc123...",
  "events": ["order.paid", "order.fulfilled"],
  "enabled": true,
  "created_at": "2026-04-09T10:00:00Z"

```

The `secret` is shown once. Store it immediately — it is used to verify signatures.

Passing an empty `events` array subscribes to all events.

## Event payload shape

```json
{
  "event_type": "order.paid",
  "data": {
    "object": "order",
    "id": "ord_9xM4kP7nR2qT5wY1",
    "listing_id": "lst_7kQ2xY9mN3pR5tW1vB8a",
    "buyer_email": "buyer@example.com",
    "amount": 1900,
    "currency": "USD",
    "status": "paid",
    "has_deliverables": false,
    "listing_metadata": {},
    "checkout_data": {"note": "Gift wrap please"},
    "fulfill_url": "/v1/orders/ord_9xM4kP7nR2qT5wY1/fulfill",
    "paid_at": "2026-04-09T14:30:00Z"
  }
}
```

`data` contains the full resource object. For order events, this is the complete order — no additional API call needed.

## Verifying signatures

Every request includes a `Listbee-Signature` header with an HMAC-SHA256 signature of the raw request body, keyed with your webhook secret.

```python
import hmac
import hashlib

def verify_webhook(body: bytes, signature_header: str, secret: str) -> bool:
    expected = hmac.new(
        secret.encode(),
        body,
        hashlib.sha256,
    ).hexdigest()
    return hmac.compare_digest(expected, signature_header)

# In your handler:
body = request.body()        # raw bytes — do not parse first
sig = request.headers["Listbee-Signature"]

if not verify_webhook(body, sig, "whsec_..."):
    return 401
```

Always verify signatures before processing. Reject requests with invalid or missing signatures.

## Event types

| Event | When it fires |
|-------|--------------|
| `order.paid` | Stripe confirmed payment |
| `order.fulfilled` | Content delivered to buyer |
| `order.refunded` | Full refund processed |
| `order.disputed` | Buyer opened a Stripe dispute |
| `order.dispute_closed` | Dispute resolved |
| `order.canceled` | Order abandoned or payment failed |
| `listing.created` | Listing created |
| `listing.updated` | Listing fields updated |
| `listing.published` | Listing went live |
| `listing.out_of_stock` | Stock reached zero |
| `listing.deleted` | Listing deleted |
| `customer.created` | New buyer placed their first order |

## Retrying a failed event

<CodeBlocks>
  <CodeBlock title="Python">
```python
client.webhooks.retry_event(
    webhook_id="wh_7kQ2xY9mN3pR5tW1vB8a",
    event_id="evt_3kQ2xY9mN3pR5tW1vB8a",
)
```
  </CodeBlock>
  <CodeBlock title="curl">
```bash
curl -X POST \
  "https://api.listbee.so/v1/webhooks/wh_7kQ2xY9mN3pR5tW1vB8a/events/evt_3kQ2xY9mN3pR5tW1vB8a/retry" \
  -H "Authorization: Bearer lb_..."
```
  </CodeBlock>
</CodeBlocks>

Retrying clears `failed_at`, `last_error`, and attempt count, then enqueues immediate redelivery.

## Webhook fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique webhook ID (`wh_` prefixed) |
| `name` | string | Display label |
| `url` | string | HTTPS endpoint receiving events |
| `secret` | string | HMAC-SHA256 signing secret (`whsec_` prefixed) — shown once at creation |
| `events` | array | Subscribed event types; empty = all events |
| `enabled` | boolean | `false` when paused or auto-disabled |
| `disabled_reason` | string | `consecutive_failures` when auto-disabled |
| `readiness` | object | Actionable status if the webhook needs attention |
